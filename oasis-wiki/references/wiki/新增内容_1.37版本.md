### Tween 动画使用指南 (id:20405)

Tween 动画系统提供了一套完整的差值平滑动画能力，支持位置、旋转、颜色等多种数值类型的平滑过渡。

**注意：Tween动画只在客户端生效，相关接口也只能在客户端调用。**

#### 支持的动画类型

| 动画类型 | API 函数 | 说明 |
|---|---|---|
| Actor位置 | `UGCTweenSystem.TweenActorLocation` | Actor位置移动 |
| Actor旋转 | `UGCTweenSystem.TweenActorRotation` | Actor旋转动画 |
| 浮点数值 | `UGCTweenSystem.TweenFloatValue` | 进度条、数值显示 |
| 向量数值 | `UGCTweenSystem.TweenVectorValue` | UI元素移动 |
| 旋转器数值 | `UGCTweenSystem.TweenRotatorValue` | 角度控制 |
| 颜色数值 | `UGCTweenSystem.TweenColorValue` | 颜色渐变 |

#### 缓动类型（33种）

完整列表：Linear, QuadIn/Out/InOut, CubicIn/Out/InOut, QuartIn/Out/InOut, QuintIn/Out/InOut, SineIn/Out/InOut, ExpoIn/Out/InOut, CircIn/Out/InOut, ElasticIn/Out/InOut, BackIn/Out/InOut, BounceIn/Out/InOut

#### 动画配置

```lua
local Config = UGCTweenSystem.MakeConfig(Delay, RepeatCount, bYoyo, RepeatDelay)
-- Delay: 初始延迟(秒)
-- RepeatCount: 0=不重复, -1=无限循环, N=总共N+1次
-- bYoyo: 是否往返(A→B→A)
-- RepeatDelay: 重复间隔(秒)
```

#### 代码示例 — 颜色动画

```lua
function TweenColor:Construct()
    local StartColor = KismetMathLibrary.MakeColor(0, 1, 0, 1)  -- 绿色
    local EndColor = KismetMathLibrary.MakeColor(1, 0, 0, 1)    -- 红色
    local Callback = function(Object, Value)
        -- Object是系统自动传入的占位符, Value是当前数值
        self.TweenTarget:SetColorAndOpacity(Value)
    end
    local Config = UGCTweenSystem.MakeConfig(0, -1, false, 0)
    self.TweenHandler = UGCTweenSystem.TweenColorValue(StartColor, EndColor, 1.0, self.TweenType, Callback, Config)
    UGCTweenSystem.PauseTween(self.TweenHandler)
end
```

#### 代码示例 — Actor位置动画

```lua
function MoveActor:Construct()
    local Actor = UGCActorComponentUtility.GetActorByActorInstancePath("UGCmap.MyActor")
    local TargetLocation = UGCMathUtility.MakeVector(100, 100, 100)
    local Config = UGCTweenSystem.MakeConfig(0, 0, false, 0)
    local TweenHandler = UGCTweenSystem.TweenActorLocation(Actor, TargetLocation, 2.0, EEasingType.EaseOutQuad, Config)
end
```

#### 控制动画

```lua
-- 暂停/恢复
UGCTweenSystem.PauseTween(TweenHandler)
UGCTweenSystem.ResumeTween(TweenHandler)
-- 动态修改配置（延迟/循环/往返）
UGCTweenSystem.ConfigureTween(TweenHandler, 0, -1, true, 0)
-- 停止（回到起始值）
UGCTweenSystem.KillTween(TweenHandler)
-- 链式动画
UGCTweenSystem.ChainTween(Handler1, Handler2)
-- 完成回调
UGCTweenSystem.BindCompletedDelegate(TweenHandler, function() print("动画完成！") end)
-- 验证动画状态
UGCTweenSystem.IsTweenValid(TweenHandler)
```

## 进阶内容

### 交互物 (id:20416) — 🔥 ActivityBaseActor

交互物继承自 `ActivityBaseActor` 基类，提供状态机、带同步的Sequence、网络同步优化等功能。

#### 创建交互物蓝图

创建继承自 `ActivityBaseActor` 的蓝图类。

#### 配置状态机

在属性【StateMachineInfo】中添加状态（默认自带"Normal"状态）：

| 属性 | 说明 |
|---|---|
| StateName | 状态名称 |
| LinkArray | 状态切换链接 |
| StateMachineEntry | 是否为入口状态（必须仅一个） |
| SequenceType | 动画序列配置 |
| Loop | 播放循环方式（-1=无限,0=一次,1=两次） |
| PlayRate | 播放速度 |
| ShouldPauseAtLastFrame | 是否在最后一帧暂停 |

LinkArray配置：
| 属性 | 说明 |
|---|---|
| LinkName | 链接名称 |
| LinkConditionFunctionName | 链接条件函数名 |
| NotCondition | 是否对条件取非 |
| LinkStateName | 目标状态名 |

#### 编写状态脚本

```lua
local MyActivityActor = {}

function MyActivityActor:Normal_Entry()
  if self:HasAuthority() then
    ugcprint("进入Normal状态（服务端）")
  else
    ugcprint("进入Normal状态（客户端）")
  end
end

function MyActivityActor:Normal_Exit()
  if self:HasAuthority() then
    ugcprint("退出Normal状态（服务端）")
  else
    ugcprint("退出Normal状态（客户端）")
  end
end

return MyActivityActor
```

也可以覆写 `OnEnterState_BP(StateName)` 和 `OnLeaveState_BP(StateName)`。

#### 相关功能组件

| 组件 | 说明 |
|---|---|
| ActivityFakePossessComponent | 交互时Possess另一个Actor |
| ClickActorComponent | 交互按钮显示组件（ForceClient属性可即时响应） |
| CustomActorMoveComponent | 为Actor提供简单移动功能 |

#### 注意事项 — 状态原子性

每个状态必须原子：状态A的Entry内创建的UI，必须在A_Exit时销毁，不能依赖客户端一定进入状态B。客户端可能丢失中间状态。

#### 网络同步与裁剪

- **网络剔除距离平方**：控制交互物进行网络同步的距离（单位cm²，值=距离×距离）
- **网络裁剪**：勾选【Region Based Net Consideration】，配置裁剪距离(Small=4000cm, Medium=15000cm, Large=35000cm, SuperLarge=100000cm)
- **变量同步**：ActivityBaseActor标记为LazyReplication，需按需同步：

```lua
function MyActivityActor:GetReplicatedProperties()
    return {"bIsOpen", "CurrentHealth"}
end

function MyActivityActor:SetOpen(bOpen)
    self.bIsOpen = bOpen
    UnrealNetwork.RepLazyProperty(self, "bIsOpen")
end
```

| UActivityFakePossessComponent | 对象函数 | 将Actor控制权传递给玩家的组件 |

## 性能优化及异常处理

### 网络相关优化 (id:20410) — 🔥 怪物距离分档网络优化

核心思路：**离玩家越远的怪物，同步精度越低**。

#### 一、网络复制基础参数

| 配置项 | 说明 | 默认值 | 推荐范围 |
|---|---|---|---|
| NetUpdateFrequency | 每秒同步次数 | 20 Hz | 5~20 Hz |
| MinNetUpdateFrequency | 无变化时最低频率 | 1.0 Hz | 0.5~2 Hz |
| NetCullDistanceSquared | 完全不同步的距离(cm²) | 22,500,000 | 按场景调整 |

**常用剔除距离值**：
| 期望距离 | 应填入值 |
|---|---|
| 50米 | 25,000,000 |
| 80米 | 64,000,000 |
| 100米 | 100,000,000 |
| 150米 | 225,000,000 |
| 200米 | 400,000,000 |

怪物类型推荐频率：
| 怪物类型 | 推荐值 |
|---|---|
| 近战Boss/高速怪 | 15~20 Hz |
| 普通巡逻怪 | 5~10 Hz |
| 远程站桩/低速NPC | 2~5 Hz |
| 装饰性背景NPC | 1~2 Hz |

#### 二、距离分档网络优化（核心推荐）

在怪物属性面板 **Network Optimization** 分类中开启：
- 启用距离分档网络优化
- 距离检测间隔（默认0.5秒）

**标准三档配置（推荐，100以内怪物）**：

| 档位 | 距离阈值(cm) | 更新频率 | Tick间隔 | 平滑最大距离 | 不平滑距离 | 平滑模式 | 碰撞模式 |
|---|---|---|---|---|---|---|---|
| 近 | 3000(30m) | 20Hz | 0.033s | 400 | 1000 | 指数 | 查询和物理 |
| 中 | 8000(80m) | 8Hz | 0.15s | 1000 | 3000 | 线性 | 仅查询 |
| 远 | 20000(200m) | 2Hz | 0.5s | 5000 | 10000 | 关闭 | 无碰撞 |

**大规模场景配置（200+怪物，塔防等）**：

| 档位 | 距离阈值(cm) | 更新频率 | Tick间隔 | 平滑最大距离 | 不平滑距离 | 平滑模式 | 碳撞模式 |
|---|---|---|---|---|---|---|---|
| 近 | 1500(15m) | 15Hz | 0.05s | 500 | 1500 | 指数 | 查询和物理 |
| 中近 | 4000(40m) | 6Hz | 0.2s | 1500 | 4000 | 线性 | 仅查询 |
| 远 | 10000(100m) | 2Hz | 0.5s | 5000 | 10000 | 关闭 | 无碰撞 |

#### 配置步骤

1. 在实体编辑器中选中怪物Actor
2. 属性面板找到 **Replication** 分类
3. 设置NetUpdateFrequency（普通怪物建议10）
4. 设置MinNetUpdateFrequency为1.0
5. 设置NetCullDistanceSquared（按场景大小）
6. 找到 **Network Optimization** 分类
7. 勾选"启用距离分档网络优化"
8. 添加档位（**按距离从小到大排列**）

#### 性能收益参考（100怪物PVE场景）

| 指标 | 优化前 | 优化后 | 提升 |
|---|---|---|---|
| 客户端Tick开销 | 基线 | 降低40%~60% | 显著 |
| 网络带宽 | 高 | 明显降低 | 较大 |
| 玩家体验 | 卡顿 | 流畅 | 明显改善 |

特惠礼包模板是在原有礼包模板基础上新增的扩展功能，增加**游戏外售卖、游戏内发货**的能力。

## AI工具

### UGCAskQ MCP 使用说明 (id:20414) — 🔥 AI+MCP协作接口

> ⚠️ 实验性功能，使用前请备份工程！
> 最后更新: 2026-07-10（官方大幅扩充）

#### 概述

UGCAskQ MCP（Model Context Protocol）是绿洲启元编辑器的 AI 协作接口，允许 Claude、Cursor 等 AI 助手直接读取和操作编辑器中的各类内容，让开发者可以通过自然语言来完成复杂的编辑器操作。

连接后，AI 助手能够感知当前编辑器的状态（当前选中了哪些对象、场景里有什么、哪个技能配置了什么），并代替开发者执行操作，例如批量添加 Actor、配置技能属性、生成行为树节点等。

> **推荐做法：** 使用更强的大语言模型，可以获得更准确、更流畅的操作体验。

#### 快速接入

**第一步：打开 MCP Server**

编辑器菜单 **窗口→开发者工具→MCP Server**，打开 MCP Server 控制面板。若显示 **Running（Port: 33444）**（绿色），说明服务已正常启动。若未启动，点击 **Start Server**。

> 如果端口连接失败，可以在面板中修改 Port 值后重新点击 Start Server。

**第二步：在 AI 客户端中配置 MCP**

以 Claude 为例，在项目根目录创建 `.mcp.json`：

```json
{
  "mcpServers": {
    "ugcaskq": {
      "type": "sse",
      "url": "http://127.0.0.1:33444/sse"
    }
  }
}
```

将 `33444` 替换为实际端口号。保存后重启 AI 客户端。

> **Cursor / VS Code 用户：** 在对应客户端的 MCP 设置中，以 SSE 类型添加上述地址即可。

**第三步：验证连接**

向 AI 助手发送："当前场景里有哪些 Actor？" 或 "你现在能在编辑器里做哪些操作？"

#### MCP Server 控制面板说明

| 配置项 | 说明 | 推荐设置 |
|---|---|---|
| Port | 服务监听端口，默认 33444 | 默认即可，冲突时修改 |
| Enforce Level | PRV 安全策略级别 | `Observe`（推荐，警告但不阻断） |
| Enable MCP Call Logging | 是否记录每日调用日志 | 开启 |
| Idle Boundary (sec) | 空闲超时阈值，超过后开启新的任务段 | 90 秒（0 表示关闭） |

点击 **Open Log Folder** 可查看日志目录：`Saved/log/MCP_YYYYMMDD.log`

#### 支持的编辑器功能（9大模块）

**1. 场景与 Actor 管理**
- 创建和放置 Actor，指定位置、旋转和缩放
- 查询和修改 Actor 属性（Transform、标签、父子关系）
- 批量操作

示例：*"在坐标 (0, 0, 100) 处放置一个 BP_Rock，缩放设置为 2"*

**2. 技能编辑器**
- 创建新技能，指定类型和基本属性
- 读取现有技能配置
- 配置基础数值参数（冷却时间、消耗值等）

示例：*"帮我创建一个近战技能，冷却时间 3 秒，消耗能量 20"*

**3. 物品编辑器**
- 创建物品，设置类型、名称、图标
- 查询物品列表，按类别筛选
- 修改数值属性

示例：*"创建一个消耗品类型的物品，名称为'急救包'，描述为'恢复 50 点生命值'"*

**4. 行为树**
- 创建和读取行为树资产
- 添加和配置行为树节点（Selector、Sequence、Task 等）
- 为节点挂载 Lua 脚本

示例：*"帮我给 Boss 行为树添加一个巡逻节点，当血量低于 30% 时切换到逃跑状态"*

**5. 蓝图编辑器**
- 创建新蓝图资产，指定父类
- 查询蓝图变量和函数列表
- 添加变量（Bool、Int、Float、Vector 等）
- 向黑板中添加键值

示例：*"在角色蓝图中添加一个名为 'IsAttacking' 的 Bool 变量"*

**6. UI 编辑器（UMG）**
- 创建 UMG 蓝图资产
- 添加控件（Button、Text、Image、ProgressBar 等）
- 设置控件位置、大小和锚点
- 配置文本内容和颜色

示例：*"创建一个血量条 Widget，包含一个 ProgressBar 和一个显示数字的 Text 控件"*

**7. 数据表（配置表）**
- 读取数据表行数据
- 添加和修改数据行
- 支持物品表、技能表、掉落表

示例：*"在物品配置表中添加一行，ID 为 1001，类型为武器，攻击力 50"*

**8. 实体编辑器**
- 查询实体当前配置
- 修改实体属性（生命值、移速、攻击距离等）
- 读取实体挂载的组件信息

示例：*"把精英小怪的生命值改为 800，移速提高 20%"*

**9. 资产查找与视窗控制**
- 搜索资产：按名称或类型在内容浏览器中查找
- 视窗操作：控制摄像机位置和朝向，聚焦指定 Actor

示例：*"帮我找到名称包含 'Weapon' 的所有静态网格资产"*

#### 推荐使用方式

1. **批量内容生成**：按坐标列表批量放置同类 Actor
2. **技能原型快速迭代**：自然语言配置技能参数
3. **配置核查与批量修改**：读取所有同类技能，汇总参数
4. **UI 布局辅助**：描述界面样式，AI 搭建控件结构
5. **行为树搭建**：描述 NPC 行为逻辑，AI 添加和连接节点

#### 常见问题

**Q: AI 连接后说"找不到服务器"？**
A: 确认 MCP Server 控制面板显示为 Running 状态；检查 `.mcp.json` 端口号与面板一致；重启 AI 客户端。

**Q: 操作后编辑器没有反应？**
A: 部分操作需手动刷新对应面板。例如蓝图变量添加后，需要重新打开蓝图编辑器。

**Q: 如何让 AI 了解当前场景状态？**
A: 直接询问 AI "当前场景里有什么？"或"我现在选中了什么？"，AI 会通过 MCP 查询编辑器状态。

**Q: 调用记录在哪里查看？**
A: 点击面板中的 **Open Log Folder**，查看 `MCP_YYYYMMDD.log`。

**Q: 能不能一边操作一边让 AI 辅助？**
A: 可以。AI 会感知当前选中的 Actor 或打开的编辑器，基于操作上下文提供帮助。

#### 注意事项

- MCP Server 仅接受来自 **本机** 的连接，不支持远程访问
- AI 操作效果直接反映在编辑器中，**建议操作前先保存工程**
- 复杂多步骤操作建议分阶段执行，每阶段确认效果后再继续
- 部分编辑器（特效编辑器、音频编辑器等）当前暂无 MCP 接口

### 导航网格 (id:20426)

怪物移动依赖寻路系统，通过导航网格生成可移动区域，AIWorldVolume赋予识别交互对象能力。

**关键步骤**：
1. 放置 `Nav Mesh Bounds Volume`，缩放覆盖可移动区域
2. 点击工具栏 `构建` 按钮烘焙导航网格（按P开关查看）
3. 放置 `AI World Volume`，缩放覆盖整个关卡（一个关卡仅能放一个）
4. UGCGameMode 蓝图配置：
   - Component Manager 添加 `ESS_Navigation` + `ESS_MonsterCrowdMove` 子系统
   - DataManager 添加 `GMData Source Navigation`
5. 确认世界设置中"游戏模式重载"与当前 UGCGameMode 一致
6. 编译保存

> 若怪物仍不能正常移动，重载 UGCGameMode 避免引擎底层 bug 导致游戏模式未正确生效。
