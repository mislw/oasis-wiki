## 开启/禁用所有伤害

> 文档路径: 常用实战教学 > 开启/禁用所有伤害

**涉及API:** `Action_SetEnableDamage`, `Execute`, `UGCGameSystem`

# 开启/禁用所有伤害

- 在GameModez中设置EnableDamage

![001.00a5c143[1].png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/001.00a5c143%5B1%5D.png)

- 代码中修改示例

```lua
local Action_SetEnableDamage = 
{
	IsEnableDamage = false;
}


function Action_SetEnableDamage:Execute()
	ugcprint(string.format("Action_SetEnableDamage:Execute IsEnableDamage[%s]", tostring(self.IsEnableDamage)));

	UGCGameSystem.GameMode.bEnableDamage = self.IsEnableDamage;

	return true;
end


return Action_SetEnableDamage
```

# 3D坐标转换屏幕2D坐标

- 目标Location在移出目标后不会显示，Success将为false
``` lua
local Location = {0,0,0} --Location为需要转换的3D坐标
local SizeX,SizeY = PlayerController:GetViewportSize()
local Scale = WidgetLayoutLibrary.GetViewportScale(GameFrontendHUD)
local Success,ScreenPos = PlayerController:ProjectWorldLocationToScreen(Location)
ScreenPos.x = (ScreenPos.x - SizeX/2) / Scale
ScreenPos.y = (ScreenPos.y - SizeY/2) / Scale
```


- 目标Location移出屏幕后返回边缘的ScreenPos

``` lua
local Limit = {X=10, Y=100, Z=10, W=150}
local ScreenPos, Angle = STExtraBlueprintFunctionLibrary.GetScreenLocationOfWorldLoc(WorldContextObject,Location,Limit)
```
- Limit参数是指和边界的间隔

## 添加切换颜色的代码

- 给Actor的Box添加仅客户端触发的代码，当角色进入Box范围内，触发修改材质颜色

``` lua

---@field DefaultSceneRoot USceneComponent

--Edit Below--

local TestCube = {};

TestCube.DynamicMaterial = nil



function TestCube:ReceiveBeginPlay()

    self.SuperClass.ReceiveBeginPlay(self);

    

    if not UGCGameSystem.IsServer() then

        --创建材质实例

        local MaterialInst = UE.LoadObject(UGCGameSystem.GetUGCResourcesFullPath('Asset/Blueprint/TestCubeMaterial.TestCubeMaterial'))

        self.DynamicMaterial = self.Cube:CreateAndSetMaterialInstanceDynamicFromMaterial(0, MaterialInst)



        self.Box.OnComponentBeginOverlap:Add(self.Box_OnComponentBeginOverlap,self)

    end

end



function TestCube:Box_OnComponentBeginOverlap(OverlappedComponent, OtherActor, OtherComp, OtherBodyIndex, bFromSweep, SweepResult)

    ugcprint("TestCube:Box_OnComponentBeginOverlap")

    --替换颜色为黄色

    self.DynamicMaterial:SetVectorParameterValue("Color_Modifier",{R = 1, G = 1, B = 0, A = 0})

end

```

## 玩家出生配置

在属性组【Respawn】中，选择并配置玩家出生位置的规则，各类玩家出生方式说明如下：

|选项名称|说明|
|-|-|
|使用默认的PlayerStart作为出生点|工程的默认出生方式，该选项会将关卡中第一个找到的 PlayerStart点作为所有玩家的出生位置|
|使用指定世界坐标作为出生点|将属性【指定出生世界坐标】作为所有玩家的出生位置<br>![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/EEn5fimage.png)|

## 刀刃球同步算法

> 文档路径: 常用实战教学 > 刀刃球同步算法

# 刀刃球同步算法

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/k8W9vimage.png)

+ 球的位置由两个数据决定，1 贝塞尔曲线，2 飞行时间。通过飞行时间可以计算出这个时间点球在贝塞尔曲线上的位置。

+ 击球时：DS端确定贝塞尔曲线+击球时间戳，通过RPC通知Client。DS和Client同时以击球时间戳模拟球飞行运动。

+ 飞行过程：DS定期推送当前飞行时间。Client收到后对比自己模拟的时间，如果位置不同超过一定距离，则以DS推送的当前时间计算出新的位置。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9ruFqimage.png)
