# 通用掉落表

游戏中通常有很多随机生成单个物品或者多个物品集的场景，例如怪物掉落物资、任务获得随机奖励、[宝箱抽奖](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20027) 等等，一般通过⌜掉落⌟的概念来实现这类功能。

<br>

## 掉落概述

掉落本质上是一种随机过程，通过预设的物品库和随机规则，从物品库中随机抽取并返回符合规则数量的物品，只要涉及随机产出物的场景都能使用掉落来处理。

![QQ20250107110056-ezgif.com-video-to-gif-converter.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/UH9f0QQ20250107110056-ezgif.com-video-to-gif-converter.gif)

为了方便开发者快速实现掉落功能，编辑器封装并提供了通用的接入方式，以满足不同场景下的掉落需求。

掉落功能的实现分为两部分：定义掉落规则及 [调用API](https://developer.gp.qq.com/api/#/searchContent/UGCDropSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCDropSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCDropSystem) 获取掉落结果，掉落规则为配表的形式，无需在脚本中硬编码实现，扩展性与灵活性更强。

<br>

## 创建配表

掉落规则表用于定义抽取物品的方式，包括抽取的数量、抽取概率等，根据掉落物结果的组合性场景分为掉落表与掉落组表两种
- 掉落表：定义一组物品集的掉落规则，规则的生效目标为每个物品
- 掉落组表：定义多个掉落表的掉落规则，规则的生效目标为每个掉落表，适用于产出多个掉落表的组合结果，例如必出A类物品+随机B类物品

### 掉落表

掉落表配置可掉落的物品组与掉落规则，每次执行掉落都从该物品组中按照规则抽取一个或多个物品，掉落规则包括掉落次数（随机次数）及掉落概率。

> 创建方式：点击编辑器菜单栏 ``窗口 -> Create System Default Table -> Create Drop and DropGroup Table``，创建的掉落表将自动添加到 ``Asset/Data/Table`` 目录下

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/I3NXpimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/sOkvGimage.png)

- 掉落ID：标识该掉落表的唯一的ID
- 描述：该掉落的业务描述，方便表格管理使用
- 掉落类型：
	- 概率：每个物品的掉落概率为 **``预设概率值/10000``**，各物品独立计算概率，每次随机结果可能产出多个物品，也可能无任何产出
	- 权重：每个物品的掉落权重为 **``预设物品权重/权重之和``**，各物品间权重互斥，保证每次随机一定有且仅有一个结果
- 随机次数：执行一次掉落会进行随机的次数
- 物品信息：参与一次随机的所有物品信息
	- 权重/概率：填写物品的概率或者权重值，对于必定掉落的物品概率/权重为10000
	- 物品ID：支持 [虚拟物品](https://developer.gp.qq.com/wikieditor/#/catalog/20139) 和物资系统中的实体物品，不区分物品的业务含义
	- 物品数量Min：若最终掉落了该物品，掉落数量的最小值
	- 物品数量Max：若最终掉落了该物品，掉落数量的最大值

**配置示例**

在掉落表（UGCDrop）中插入一条数据，掉落ID设置为1，一共配置了三个物品：1001、1002与1003。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/EYFl9image.png)

示例中通过 ```概率``` 的方式进行掉落，由于物品1001的概率值为10000，因此必定掉落该物品，而物品1002和物品1003非必定掉落，取决于随机结果。

---

### 掉落组表

掉落组表允许多个掉落表结果的组合，先根据掉落组设定的规则随机出一个掉落ID，再基于该掉落ID对应表的随机规则确定最终掉落物。

> 创建方式：点击编辑器菜单栏 ``窗口 -> Create System Default Table -> Create Drop and DropGroup Table``，创建的掉落组表将自动添加到 ``Asset/Data/Table`` 目录下

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/I3NXpimage.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1NLGNimage.png)

- 掉落组ID：该掉落组的唯一ID
- 描述：掉落组的业务描述，方便表格管理使用。
- 掉落类型：规则同掉落表，区别为最终随机出的是掉落表ID，而非物品ID
- 掉落组信息：参与一次随机的掉落组信息
	- 权重/概率：填写掉落表的概率或者权重值，对于必定掉落的掉落表概率/权重为10000
	- 掉落ID：本次掉落出的掉落表ID

**配置示例**

创建三张掉落表，分别控制枪械、防具以及Buff的掉落，掉落表ID分别为1、2、3。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1cugqimage.png)

在掉落组表（UGCDropGroup）中插入一条数据，对三类物品进行组合随机。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/VOs9Aimage.png)

掉落类型设置为 ```概率```，枪械掉落表的概率值为10000，因此掉落结果必定包含枪械类物品，而是否掉落防具或者Buff则取决于随机结果。

## 获取掉落结果

针对掉落表和掉落组表分别提供了 [DropItems](https://developer.gp.qq.com/api/#/searchContent/UGCDropSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCDropSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCDropSystem&autoJump=DropItems) 和 [DropItemsByGroup](https://developer.gp.qq.com/api/#/searchContent/UGCDropSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCDropSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCDropSystem&autoJump=DropItemsByGroup) 两个API用于获取掉落结果。

以掉落表的随机结果为例，以下为具有三种掉落物品的掉落表配置：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Lw2Tbimage.png)

在触发执行掉落的场景中调用 [``DropItems``](https://developer.gp.qq.com/api/#/searchContent/UGCDropSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCDropSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCDropSystem) 接口以获取掉落结果：

``` lua
  function UGCPlayerController:TestHandle()

    for k,v in pairs(DropResult) do
        ugcprint("k：" .. tostring(k) .. " " .. "v：" .. tostring(v))
    end
end
```

通过打印日志可以查看最终获得的掉落物品list，key值为物品ID，value值为该类物品的总数量。

## 生成掉落物

当掉落表中的物品为实体物品时，可于脚本中调用 [``StartCustomizeDrop``](https://developer.gp.qq.com/api/#/searchContent/UGCItemSystemV2?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%89%A9%E5%93%81%E4%B8%8E%E8%83%8C%E5%8C%85%2FUGCItemSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCItemSystemV2&autoJump=StartCustomizeDrop) 函数在指定的位置生成随机的掉落物，掉落的表现效果与 [掉落组件](https://developer.gp.qq.com/wikieditor/#/catalog/20142?autoJump=%E6%A1%88%E4%BE%8B%E6%BC%94%E7%A4%BA) 相同，但不依赖于掉落组件的配置。

> 产生掉落的主体对象必须为 ``UGCDropActor_BP`` 的派生蓝图

**示例代码**

```lua
local CurrentPlayerPawn = UGCGameSystem.GetPlayerPawnByPlayerController(self)
local CurrentLocation = CurrentPlayerPawn:K2_GetActorLocation()
local DropLocation = UGCMathUtility.AddVector(CurrentLocation, Vector.New(200,0,0))
UGCItemSystemV2.StartCustomizeDrop(DropLocation, DropID, -1, EUGCGenerateItemEntityType.GenerateItemEntity_WrapperActor, CurrentPlayerPawn)
```

# GamePart模块化

GamePart通过模块化的形式将GamePlay功能解耦，并提供可选的启用配置方式，帮助开发者在易于调试与维护的前提下，为玩法灵活引入不同的游戏功能。

<br>

## GamePart概述

编辑器既有的GamePlay系统功能的实现方式耦合性强、维护成本高，因此以可复用、易扩展为原则对功能进行封装，并提供简单的模块化配置方式，这种形式称为GamePart模块化。目前已有多种功能经过GamePart封装，例如 [商城模板](https://developer.gp.qq.com/wikieditor/#/catalog/20026)、[排行榜模板](https://developer.gp.qq.com/wikieditor/#/catalog/20048)、[背包系统](https://developer.gp.qq.com/wikieditor/#/catalog/20104) 等等，开发者可按需配置启用且支持调用GamePart的内核接口。

以商城模板为例，商城涉及商业化购买流程与虚拟物品的管理逻辑，实现上对应封装了“商业化管理”与“虚拟物品管理”两个GamePart模块，配置商城模板时显式启用 ``GP_CommodityOperationManager`` 将自动加载两个模块；除了模板自身功能外，两个模块分别以 [``CommodityOperationManager``](https://developer.gp.qq.com/api/#/searchContent/CommodityOperationManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FCommodityOperationManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=CommodityOperationManager) 与 [``VirtualItemManager``](https://developer.gp.qq.com/api/#/searchContent/VirtualItemManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FVirtualItemManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=VirtualItemManager) 类函数的形式提供了对外接口，开发者可以调用API实现更加丰富的功能。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/8WtPjimage.png)

未来持续有更多GamePlay功能以模块化的方式提供功能封装。

## GamePart接口调用

各模块功能通过 ``AUGCGamePartGlobalActor`` 的派生类提供开放接口，调用模块功能的API前先通过 ``GamePartName`` 获取GamePartGlobalActor的实例对象，继而调用相关类函数。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/tUIjuimage.png)

以查询商业化中商品表的配置为例，需要前置启用 ``GP_CommodityOperationManager`` 模块，并且调用 [``GetGamePartGlobalActor``](https://developer.gp.qq.com/api/#/searchContent/UGCGamePartSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FUGCGamePartSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGamePartSystem&autoJump=GetGamePartGlobalActor) 获取 [``CommodityOperationManager``](https://developer.gp.qq.com/api/#/searchContent/CommodityOperationManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FCommodityOperationManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=CommodityOperationManager) 实例对象，再调用 [``GetAllProductData``](https://developer.gp.qq.com/api/#/searchContent/CommodityOperationManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FCommodityOperationManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=CommodityOperationManager&autoJump=GetAllProductData) 获取到商品表的配置数据。

**示例代码**

```lua
-- 获取GamePartGlobalActor的实例对象
local GamePartActorInstance = UGCGamePartSystem.GetGamePartGlobalActor("CommodityOperationManager");

-- 调用模块GamePart接口获取商品配置信息
local CommodityConfigData = GamePartActorInstance:GetAllProductData();
log_tree(CommodityConfigData)
```

## 注意事项

不同GamePart的加载时机存在差异，``ReceiveBeginPlay`` 中可能还未完成GamePart的加载，导致调用或者绑定GamePartGlobalActor实例对象的函数和委托失败，推荐做法是额外监听 [``UGCGenericMessageSystem.Messages.UGC.GamePart.GamePartLoaded``](https://developer.gp.qq.com/api/#/searchContent/UGCGenericMessageSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCGenericMessageSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGenericMessageSystem) 全局事件，在事件触发时执行相应逻辑。

**示例代码**

```lua
function UGCGameState:ReceiveBeginPlay()
		local GamePartActorInstance = UGCGamePartSystem.GetGamePartGlobalActor("VirtualItemManager");
  	if GamePartActorInstance then
    		GamePartActorInstance.AddItemResultDelegate:Add(self.OnAddVirtualItemResult, self)
  	else
    		UGCGenericMessageSystem.ListenGlobalMessage(self, UGCGenericMessageSystem.Messages.UGC.GamePart.GamePartLoaded, self, self.OnGamePartLoaded)
  	end
end

function UGCGameState:OnGamePartLoaded(GamePartName)
    if GamePartName == 'VirtualItemManager' then
        local GamePartActorInstance = UGCGamePartSystem.GetGamePartGlobalActor("VirtualItemManager");
        GamePartActorInstance.AddItemResultDelegate:Add(self.OnAddVirtualItemResult, self);
    end
end

function UGCGameState:OnAddVirtualItemResult()
	-- 当获得虚拟物品时执行的逻辑
end
```

# 通用掉落组件

玩法中应用掉落的场景非常广泛，包括击杀怪物掉落、打开宝箱掉落、资源采集掉落等等，通用掉落组件通过子组件的形式提供为游戏对象赋予掉落的能力，开发者仅需为游戏对象添加该组件并配置掉落规则即可主动触发掉落，同时组件自身高复用的特点也带来开发效率提升的优势。

<br>

## 添加掉落组件

掉落组件是一个名为“UGCPresetCommonDropItemComponent”的 [Actor组件](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/components-in-unreal-engine)，通过目标对象蓝图的【添加新组件】按钮搜索并添加该组件。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wjb6Aimage.png)

<br>

## 配置掉落组件

### 掉落物生成规则

针对掉落物的生成规则配置，本组件提供掉落表或者蓝图配置两种可选模式。

> 掉落组件只支持实体物品的掉落，因此配置中相关的物品ID均需为物资系统的ID

**掉落表**

基于 [通用掉落表](https://developer.gp.qq.com/wikieditor/#/catalog/20058) 配置的规则执行掉落，组件通过掉落表ID进行关联。执行掉落时会根据对应表内的概率/权重配置，掉落物种类的数量随机。

在该组件属性中搜索“Drop item Generator Config”，将“列表生成方式”选择为读表，并将配置好的掉落表ID或者掉落组表ID填入对应的输入框中。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/EUSQnimage.png)

- 掉落方案ID：对应UGCDrop数据表中的掉落ID
- 掉落组方案ID：对应UGCDropGroup数据表中的掉落组ID，优先级高于掉落方案ID
- 掉落物类型：
	- Wrapper：物品将以可拾取物的形式掉落在场景中，玩家需通过拾取操作将其收入背包
	- 背包：掉落物直接进入玩家背包，兼容经典背包与新背包系统

**蓝图配置**

在组件属性中配置需要掉落的物品信息，该方式掉落效果为固定掉落，掉落物种类的数量固定。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1KRYgimage.png)

- Item ID：需要进行掉落的目标物品ID
- Count：掉落数量
- 掉落物类型：
	- Wrapper：物品将以可拾取物的形式掉落在场景中，玩家需通过拾取操作将其收入背包
	- 背包：掉落物直接进入玩家背包，兼容经典背包与新背包系统

---

### 掉落点生成规则

掉落点的生成方式为圆环形，所有掉落点均按照规则随机分布于由内径和外径两个同心圆所界定的环形区域内。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/e41dximage.png)

- 掉落起点偏移：以掉落组件的原始坐标为基准，通过预设的偏移量进行位置调整，从而确定最终的掉落起始点
- 掉落圆环偏移：以初始圆环的中心点为基准，按照预设的偏移量调整整个圆环的位置，从而改变所有掉落终点的最终分布区域
- 掉落范围内径：掉落圆环的最小半径，决定了掉落区域的内侧边界，掉落点不会出现在比内径更靠近圆心的范围内
- 掉落范围外径：掉落圆环的最大半径，决定了掉落区域的外侧边界，掉落点不会出现在比外径更远离圆心的范围内
- 掉落方向：
	- 完全随机：掉落角度在360°范围内完全随机生成，确保每个掉落点的方位角独立且均匀分布，适用于需要无规律散落的场景。
	- 伪随机：掉落点按四个象限依次循环生成，每个象限覆盖90°范围，使掉落分布相对均匀，避免过度集中。
	- 面向玩家：掉落方向以玩家当前位置为基准，限定在玩家朝向的正负45°区域内，适用于需要指向性投放的场景。
- 随机的角度范围：仅适用于掉落方向为完全随机掉落方式，可以约束完全随机的掉落角度范围

---

### 掉落实体表现

该选项只有当掉落物类型选择为 ``Wrapper`` 时生效，可以设置拾取物的生命周期和是否显示掉落特效。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/hvaxzimage.png)

- Wrapper的存在时长：控制掉落物Wrapper实体的存续时间，单位为秒
- 是否需要特效表现：是否需要在物品掉落时展示相应的掉落特效动画

---

### 掉落波次规则

执行掉落时，如果设置的每波掉落次数小于掉落物种类的数量，就会分多批掉落，每次掉出一种物品且不同类型的物品掉出的波次顺序随机。例如掉落5种物品，每波掉落次数为2，则会分3波掉落，前2波各掉落2次（即各生成2种掉落物），第3波掉落剩余的1种物品。

在该组件属性中搜索“Step”，即可进行相关配置。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/fs0uFimage.png)

- 每波掉落次数：执行掉落时，每一波的掉落次数，每次掉落一种物品，掉落物类型为Wrapper时生效
- 每波掉落间隔：相邻两波掉落之间的时间间隔，单位：秒，掉落物类型为Wrapper时生效
- 起始掉落延迟：从执行掉落到生成第一波掉落物的延迟时间，单位：秒

<br>

## 执行掉落

掉落组件提供了 [``StartDrop``](https://developer.gp.qq.com/api/#/searchContent/UUGCCommonProduceDropItemComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUGCCommonProduceDropItemComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUGCCommonProduceDropItemComponent&autoJump=StartDrop) 与 [``StartDropByProduceID``](https://developer.gp.qq.com/api/#/searchContent/UUGCCommonProduceDropItemComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUGCCommonProduceDropItemComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUGCCommonProduceDropItemComponent&autoJump=StartDropByProduceID) 函数用于触发执行掉落行为，开发者在脚本中于合适的时机调用该函数即可实现掉落。

<br>

``` lua
function TreasureZombie:PostTakeDamageEvent(Damage, EventInstigator, DamageCauser, DamageContext)
```

**交互物掉落**

交互物的掉落通常需要在指定的时机下触发，例如宝箱在玩家点击打开UI时触发，下面设计一个简单的案例演示实现过程：当玩家角色移动经过特定区域时，自动将预设物品添加到玩家背包中，实现移动中拾取的效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Lyshvimage.png)

配置掉落点生成方向为“完全随机”，掉落范围设置为内径100和外径200的同心圆，且限制Y方向上的随机角度范围为360：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RwTy4image.png)

于服务端绑定盒体组件的 [``OnComponentBeginOverlap``](https://developer.gp.qq.com/api/#/searchContent/UPrimitiveComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUPrimitiveComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UPrimitiveComponent&autoJump=OnComponentBeginOverlap) 委托，并在回调函数中调用 [``StartDrop``](https://developer.gp.qq.com/api/#/searchContent/UUGCCommonProduceDropItemComponent?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUGCCommonProduceDropItemComponent.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUGCCommonProduceDropItemComponent&autoJump=StartDrop) 接口执行掉落：

``` lua
function TextBox:ReceiveBeginPlay()
		TextBox.SuperClass.ReceiveBeginPlay(self)
    if UGCGameSystem.IsServer() then
        self.Box.OnComponentBeginOverlap:Add(self.HandleOverlap, self)
    end
end

function TextBox:HandleOverlap(OverlappedComponent, OtherActor, OtherComp, OtherBodyIndex, bFromSweep, SweepResult)
    local IgnoreActors = {self}
    local PlayerController = GameplayStatics.GetPlayerController(self, 0);
    self.UGCPresetCommonDropItemComponent:StartDrop(self, PlayerController, IgnoreActors)
end
```

当玩家角色与预设碰撞盒体发生接触时，将自动触发物品掉落并将掉落物品放入玩家背包。

# 通用消息系统

**UGCGenericMessageSystem** 是一个游戏内广播消息处理的接口库，旨在为开发者提供一个高效、灵活的方式来管理和传递游戏中的各种消息。通过该系统，开发者可以在不同的游戏组件间进行消息广播和接收，从而实现复杂的交互逻辑。无论是监听系统预置消息还是自定义消息，开发者都可以通过统一的接口轻松完成消息的发送和监听。

<br>

## 功能概述

**UGCGenericMessageSystem** 提供了以下核心功能：

* **对象消息广播**：发送给特定对象，只有绑定该对象的监听者能接收
* **全局消息广播**：广播给所有监听者，与具体对象无关
* **系统预置消息广播**：系统预设的消息类型，可直接监听处理

<br>

## 功能详解

### 消息类型

通用消息分为两类：`自定义消息` 与 `系统预置消息`。`系统预置消息` 由系统预定义可直接使用，用于监听游戏流程中的标准事件，如玩家进入游戏、角色生成等。`自定义消息` 由开发者注册并命名，适用于项目中的个性化逻辑。两者在使用方式上类似，但系统预置消息无需注册，自定义消息需先注册后使用。

**系统预置消息示例**

``` lua
UGCGenericMessageSystem.Messages.UGC.Player.PlayerEnter -- 玩家进入游戏
UGCGenericMessageSystem.Messages.UGC.Player.PlayerExit -- 玩家退出游戏
UGCGenericMessageSystem.Messages.UGC.PlayerPawn.PostTakeDamage -- 玩家受到伤害后
……
```

更多预置消息可阅读文档：[**UGCGenericMessageSystem**](https://developer.gp.qq.com/api/#/searchContent/UGCGenericMessageSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%B7%A5%E5%85%B7%E5%BA%93%2FUGCGenericMessageSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGenericMessageSystem)

---

### 消息注册

在使用通用消息系统发送或监听自定义消息前，必须先进行消息注册。注册的过程相当于向系统声明这条消息的存在，确保它在后续的通信中可识别，从而保障消息能被正确广播与接收。

注册方法：

``` lua
UGCGenericMessageSystem.RegisterUserDefinedMessage(string Message)
```

开发者可将 `自定义消息名` 作为参数传入 `RegisterUserDefinedMessage` 方法进行注册，该方法注册时需注意 `客户端` 与 `服务端` ，消息仅在注册的一侧生效，不能跨端广播和监听，需根据实际运行环境分别注册和使用。

---

### 对象级别的消息监听
对象消息广播会把消息发送给指定对象，只有监听这个对象的逻辑才能收到，比如监听玩家金币数量的改变，通知UI更新显示。

**API**

绑定监听对象广播

``` lua
UGCGenericMessageSystem.ListenObjectMessage(ListenedObject, Message, Listener, Callback)
```

广播自定义的对象消息

``` lua
UGCGenericMessageSystem.BroadcastUserDefinedObjectMessage(ListenedObject, MessageID, ...)
```

参数详解

| 参数名              | 类型        | 说明                                           |
| ---------------- | --------- | -------------------------------------------- |
| `ListenedObject` | `UObject` | 要监听的对象，通常是游戏中的 Actor 或 Component，对象状态变化时发送消息 |
| `MessageID`      | `FName`   | 消息标识符，用于区分不同的消息类型                            |
| `Listener`       | `UObject` | 接收消息的对象，非空时会从该对象中调用回调方法，空则表示使用静态函数           |
| `Callback`       | `FName`   | 回调函数名，消息触发时会调用此方法                            |
| `ListenID`       | `number`  | 监听返回的唯一 ID，用于后续取消监听操作                        |

**使用示例**

监听玩家金币数量的改变，并将改变的金币值传递给UI显示，如下所示。

``` lua
function UGCPlayerPawn:ReceiveBeginPlay()
    UGCPlayerPawn.SuperClass.ReceiveBeginPlay(self)

    if UGCGameSystem.IsServer() then
    else
        local Message = "PlayerGoldChange"
        UGCGenericMessageSystem.RegisterUserDefinedMessage(Message) -- 注册消息
    end
end

-- 改变金币的函数
function UGCPlayerPawn:ChangeHealth()
    local Message = "PlayerGoldChange"
    UGCGenericMessageSystem.BroadcastUserDefinedObjectMessage(self, Message, 100)
end
```

首先在客户端注册消息 `PlayerGoldChange` ，并在金币改变时将消息与改变的数量一起广播，广播时 `要监听的对象` 为 `self` ，即为 `UGCPlayerPawn` ，开发者在UI控件里监听消息时，需将消息的监听对象绑定为玩家的pawn，如下所示。

``` lua
function MainUI:Construct()
    self.Button_0.OnClicked:Add(self.Handle_BtnClicked, self)

    ugcprint("注册事件")
    local Message = "PlayerGoldChange"
    local playerPawn = UGCGameSystem.GetLocalPlayerPawn()
    UGCGenericMessageSystem.ListenObjectMessage(playerPawn, Message, self, self.UpdateGoldChange)
end

function MainUI:UpdateGoldChange(gold)  
    ugcprint("更新金币: "..gold)
end
```

`MainUI` 为处理金币显示的UI控件，在此控件上监听 `PlayerGoldChange` 消息，并触发对应的回调函数，如上所示。

在 `MainUI:Construct` 中，使用 `ListenObjectMessage` 监听消息，因为监听对象为 `UGCPlayerPawn` ，所以函数中第一个参数为玩家的角色实例。如此当监听对象Pawn发送该消息时，`MainUI`控件将会接收到此条消息，并调用 `UpdateGoldChange` 方法。

### 全局级别的消息监听

全局消息广播是指将消息广播给所有监听该消息的对象，不依赖于具体对象，比如当玩家金币数量改变后，需要通知多个UI进行更新。

**API**

绑定监听全局广播消息

``` lua
UGCGenericMessageSystem.ListenGlobalMessage(WorldContextObject, Message, Listener, Callback)
```

广播自定义的全局消息

``` lua
UGCGenericMessageSystem.BroadcastUserDefinedObjectMessage(ListenedObject, Message, ...)
UGCGenericMessageSystem.BroadcastUserDefinedGlobalMessage(Message, ...)
```

对于全局消息监听，只要消息名一致，都可被广播触发，无需关注被监听对象。其中 `BroadcastUserDefinedObjectMessage` 可触发 `ListenGlobalMessage` 与 `ListenObjectMessage` 监听的事件，`BroadcastUserDefinedGlobalMessage` 只可触发全局广播消息 `ListenGlobalMessage` 监听的事件。


**使用示例**

监听玩家金币数量的改变，并将改变的金币值传递给 `MainUI` 和 `GoldUI` 同步显示，如下所示。

``` lua
function UGCPlayerPawn:ReceiveBeginPlay()
    UGCPlayerPawn.SuperClass.ReceiveBeginPlay(self)

    if UGCGameSystem.IsServer() then
    else
        local Message = "PlayerGoldChange"
        UGCGenericMessageSystem.RegisterUserDefinedMessage(Message) 
    end
end

-- 改变金币的函数
function UGCPlayerPawn:ChangeHealth()
    ugcprint("发送事件")
    local Message = "PlayerGoldChange"
    UGCGenericMessageSystem.BroadcastUserDefinedObjectMessage(self , Message, 100)
end
```

分别创建两个不同的UI控件，各自监听 `PlayerGoldChange` 事件，如下所示。

**MainUI**

``` lua

function MainUI:Construct()
    self.Button_0.OnClicked:Add(self.Handle_BtnClicked, self)

    ugcprint("注册事件")
    local Message = "PlayerGoldChange"
    local playerPawn = UGCGameSystem.GetLocalPlayerPawn()
    UGCGenericMessageSystem.ListenObjectMessage(playerPawn, Message, self, self.UpdateGoldChange)
end

    ugcprint("MainUI - 更新金币: "..gold)
end


```

**GoldUI**

``` lua
function GoldUI:Construct()
    ugcprint("注册事件")
    local Message = "PlayerGoldChange"
    UGCGenericMessageSystem.ListenGlobalMessage(self, Message, self, self.UpdateGoldChange)
end

function GoldUI:UpdateGoldChange(gold)
    ugcprint("GoldUI - 更新金币: " .. gold)
end
```

此处，`MainUI` 与 `GoldUI` 两个控件都监听 `PlayerGoldChange` 事件，不同在于 `MainUI` 监听时是针对 `UGCPlayerPawn` 对象进行监听，`GoldUI` 则是全局监听，无需特别对象。使用 `BroadcastUserDefinedObjectMessage` 发送广播后会同时触发 `MainUI` 与 `GoldUI` 两个控件的监听事件。

如果使用 `BroadcastUserDefinedGlobalMessage` 方法，发送全局广播，则只会触发 `GoldUI` 上同样使用全局监听的事件。

### 系统预置消息监听

预置消息广播是指系统内部预定义的消息类型，开发者可以通过监听和响应这些预置消息来实现特定功能，通常用于游戏框架中的标准事件处理。

``` lua
function UGCPlayerPawn:ReceiveBeginPlay()
    UGCPlayerPawn.SuperClass.ReceiveBeginPlay(self)

    if UGCGameSystem.IsServer() then
        local Message = UGCGenericMessageSystem.Messages.UGC.PlayerPawn.PostTakeDamage -- 玩家伤害事件
        UGCGenericMessageSystem.ListenObjectMessage(self, Message, self, self.HandleDamage)
    else
    
    end
end

-- 处理伤害事件
function UGCPlayerPawn:HandleDamage(VictimPlayer, DamageCauserActor, EventInstigator, Damage, DamageContext)
    ugcprint("伤害： " .. Damage)
end
```

如上所示，`UGCGenericMessageSystem.Messages.UGC.PlayerPawn.PostTakeDamage` 为预设的 `玩家伤害事件` ，开发者可在服务端监听此事件，在玩家受到伤害后，将会发送消息至接收消息的对象，进而触发回调函数 `HandleDamage` 。

其中因为事件为 `玩家伤害事件` ，所以监听的对象需为 `UGCPlayerPawn` ，即为 `ListenObjectMessage` 第一个传参。

# 通用抛体

抛体是一种基于物理模拟的具有运动行为的抛射物对象，通常以固定的轨迹不断运动且具备命中或者爆炸后的交互逻辑和效果，例如手雷等投掷物和法术飞弹等远程攻击技能都涉及抛体的应用。

技能编辑器将抛体的复杂逻辑进行高度抽象并拆解为运动轨迹、触发条件和行为效果的交互组合，封装了一套通用的参数配置化、可复用及高度扩展的抛体框架，开发者可以基于内置的抛体模板创建并快速配置实现个性化的抛射物功能。

<br>

## 抛体概述

### 基础框架

一个完整抛体的配置框架由基础属性、运动轨迹、碰撞检测、触发条件与执行动作等核心模块组成。

**基础属性**

任何抛体都有明确的生命周期，运动距离和存活时长等属性决定了此抛体何时被自动销毁，也支持通过 [GameplayTag](https://developer.gp.qq.com/wikieditor/#/catalog/20102) 标签标识抛体的类别。

**运动轨迹**

运动轨迹确定了抛体在运动过程中的方向和飞行交互行为，例如是否具备追踪特性、发生碰撞后是否继续飞行等，通常由初速度和重力系数等属性决定飞行的轨迹路径，通过物理参数和路径算法控制抛体的运动方式。

**碰撞检测**

抛体依赖碰撞的结果触发效果，因此任何抛体都必须配置一个碰撞体组件，用于检测抛体运动过程中的碰撞行为，碰撞结果可被指定的过滤条件进行筛选，以决定最终的碰撞目标。

> 碰撞体必须设置为抛体蓝图的根组件，且若无特殊需求，推荐直接使用模板里的碰撞预设

**触发条件**

抛体的生命周期内预置了一批特定事件，当事件触发时可执行指定的动作行为以实现目标效果，例如抛体与任意物体发生碰撞时产生爆炸，此时碰撞事件为条件，爆炸为执行动作，也即触发条件决定了动作的执行时机。

**执行动作**

当特定条件触发时需要执行的具体动作行为，编辑器预置了丰富的动作组件供配置使用，例如造成伤害、添加Buff、生成Actor等等。

需要注意的是，抛体的动作执行为瞬时的行为，即抛体只负责触发该动作的执行，且该动作按实际效果执行完毕，抛体不参与动作的生命周期管理，例如播放音效动作，即使抛体已销毁，音效仍会持续播放至完整时长结束。

> 如果对动作有生命周期管理的诉求，建议转为Buff进行实现

### 蓝图结构

抛体蓝图主要由 ``ProjectileMovement（抛射物）组件``、``Collision（碰撞体）组件`` 和其他粒子特效或静态网格体组成。

[抛射物组件]((https://blog.csdn.net/zxh1592000/article/details/127092319)) 模拟物体在世界场景中的运动物理状态，是抛体的核心组件。
> 此组件下的属性无需单独设置

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wB6iHimage.png)

[碰撞体组件](https://zhuanlan.zhihu.com/p/546113152) 用于检测物体是否发生碰撞以及触发特定的碰撞事件/委托，根据检测范围形状的不同分为 ``BoxCollision（立方体碰撞体）``、``CapsuleCollision（胶囊体碰撞体）`` 和 ``SphereCollision（球体碰撞体）`` 三种简单碰撞体，在抛体中必须将碰撞体组件作为与抛射物组件平级的根组件，否则在高速飞行的抛射物场景下可能出现碰撞检测失效的情况。

## 抛体配置详解

> ``启用客户端模拟`` 属性建议不勾选，但对于部分类型的飞行轨迹，勾选此选项可能有更好地表现

### 基础属性

抛体的基础属性决定生命周期，包括最大销毁距离、最大存活时长与是否忽略自己等属性。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/IQspqimage.png)

- 最大销毁距离：当抛体的运动总距离超过该值时，自动销毁抛体（单位：厘米）
- 最大存活时长：当抛体的存活时长超过该值时，自动销毁抛体（单位：秒）
- 是否忽略自己：是否忽略抛体自身的碰撞，默认勾选不建议修改，否则可能发生异常的碰撞结果

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/5oIAeimage.png)

- Gameplay Tags：基于 [GameplayTag](https://developer.gp.qq.com/wikieditor/#/catalog/20102) 的分类标签，可用于 [抛体发射事件](https://developer.gp.qq.com/wikieditor/#/catalog/20172?autoJump=%E6%8A%9B%E4%BD%93%E5%8F%91%E5%B0%84%E4%BA%8B%E4%BB%B6) 中的触发过滤条件

### 飞行轨迹

抛体必须配置一种飞行轨迹才能正常发射运动，目前已支持抛物线、追踪、目标弹射及穿透等轨迹类型。

#### 抛物线轨迹

最常用的一类飞行轨迹，由初速度和重力系数决定曲线的轨迹，例如手雷抛体即是使用抛物线轨迹。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RaPvzimage.png)

- 重力系数：重力加速度的系数，系数越大，重力效应越明显
- 初始速度：发射抛体时的初速度
- 最大速度：抛体飞行的速度上限，=0代表不限制速度
- 同单位碰撞触发间隔：同一个碰撞目标重复触发碰撞事件的间隔频率，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108) 
- 旋转跟随速度一致：抛体是否一直朝向速度的方向

重力系数和速度是否受曲线影响的配置详情跳转 [使用曲线](https://developer.gp.qq.com/wikieditor/#/catalog/20209?autoJump=%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF:~:text=%E7%8E%AF%E7%BB%95%E8%BD%A8%E8%BF%B9-,%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF,-%E7%A2%B0%E6%92%9E%E8%BF%87%E6%BB%A4%E5%99%A8)

#### 追踪轨迹-UE自带

基于UE原生Homing追踪系统实现的飞行轨迹，通常适用于追踪静态目标，如果需要追踪可移动目标，建议设置加速度参数以确保追踪效果。

> 追踪目标取决于技能编辑器中 [选择目标](https://developer.gp.qq.com/wikieditor/#/catalog/20186?autoJump=%E6%8A%80%E8%83%BDTask-%E9%80%89%E6%8B%A9%E7%9B%AE%E6%A0%87) Task的选取结果

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/PpvUwimage.png)

- 初始速度：发射抛体时的初速度
- 重力系数：重力加速度的系数，系数越大，重力效应越明显
- 加速度：抛体飞行过程中的每秒加速度大小
- 最大速度：抛体飞行的速度上限，必须大于初始速度
- 同单位碰撞触发间隔：同一个碰撞目标重复触发碰撞事件的间隔频率，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108) 
- 旋转跟随速度一致：抛体是否一直朝向速度的方向

---

#### 追踪轨迹

在原生Homing追踪系统基础上特化的高级追踪轨迹，支持配置追踪部位与偏移量，且细化出线速度、转向灵敏度、吸附范围等属性以优化追踪效果。

> 追踪目标取决于技能编辑器中 [选择目标](https://developer.gp.qq.com/wikieditor/#/catalog/20186?autoJump=%E6%8A%80%E8%83%BDTask-%E9%80%89%E6%8B%A9%E7%9B%AE%E6%A0%87) Task的选取结果

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/mMGGXimage.png)

- Speed：发射抛体时的初速度
- Acceleration：抛体飞行过程中的每秒加速度大小
- MaxSpeed：抛体飞行的速度上限
- 转向灵敏度：该参数控制追踪系统的转向响应速率，基于FMath::VInterpTo实现的方向插值计算（输入当前方向、目标方向，并将转向灵敏度传入InterpSpeed参数进行插值计算）
	- 0~2：平缓转向，目标有充足时间规避
	- 2~5：快速修正轨迹，显著增加躲避难度
	- 5~10：近似瞬时转向，常规移动难以摆脱
	- 10+：直线追踪模式，瞬间完成转向
- 每秒增加转向灵敏度：转向灵敏度随时间的变化速率，不超过最大转向灵敏度
- 最大转向灵敏度：转向灵敏度的数值上限
- 吸附距离：当抛体与目标的距离小于吸附判定距离时，抛体将强制朝向目标飞行，将永久锁定该目标直至命中，即使目标后续远离超出该距离，仍维持原有追踪逻辑不变（单位：厘米）
- Socket：需要追踪的目标部位，支持部位名或者骨骼插槽名
- Offset：相对于 ``Socket`` 的位置偏移量，仅 ``Location`` 属性生效
- 同单位碰撞触发间隔：同一个碰撞目标重复触发碰撞事件的间隔频率，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108) 
- 旋转跟随速度一致：抛体是否一直朝向速度的方向

#### 目标弹射轨迹

允许在多个目标之间来回弹射的飞行轨迹，是继承自抛物线轨迹的派生类，属于对抛物线轨迹的功能扩展。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/T9L7Kimage.png)

- 搜索半径：抛体在命中目标后将在该范围内进行二次目标搜索，按指定的规则筛选新目标并发生弹射
- 最大碰撞次数：弹射的次数上限
- 目标能否重复弹射：是否允许将同一个命中对象作为新的弹射目标
- 搜索结果过滤器：基于 [碰撞过滤器](https://developer.gp.qq.com/wikieditor/#/catalog/20165?autoJump=%E5%91%BD%E4%B8%AD%E8%BF%87%E6%BB%A4%E5%99%A8) 的搜索目标规则
- 优先级类型：当搜索到多个目标时，会依据优先级算法进行排序，选择优先级最高的目标进行弹射
	- 距离最近：优先选择距离最近的目标
	- 血量最低：优先选择血量绝对值最低的目标
	- 百分比血量最低：优先选择血量百分比最低的目标
	- 自定义方法：开发者通过覆写 [``SearchTargetActorByPriorityBP``](https://developer-test.gp.qq.com/api/#/searchContent/APESkillProjectileBase?classDetailShow=true&path=class%2Fdetail%2FOthers%2FAPESkillProjectileBase.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=APESkillProjectileBase&autoJump=SearchTargetActorByPriorityBP) 逻辑自定义实现排序规则
- 优先级算法取反：是否对优先级算法的结果取反
- 是否持续追踪目标：弹射至新目标时，是仅在弹射瞬间修正方向，还是在整个弹射过程中持续追踪该目标
- 重力系数：重力加速度的系数，系数越大，重力效应越明显
- 初始速度：发射抛体时的初速度
- 最大速度：抛体飞行的速度上限
- 同单位碰撞触发间隔：同一个碰撞目标重复触发碰撞事件的间隔频率，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108) 
- 旋转跟随速度一致：抛体是否一直朝向速度的方向

重力系数和速度是否受曲线影响的配置详情跳转 [使用曲线](https://developer.gp.qq.com/wikieditor/#/catalog/20209?autoJump=%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF:~:text=%E7%8E%AF%E7%BB%95%E8%BD%A8%E8%BF%B9-,%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF,-%E7%A2%B0%E6%92%9E%E8%BF%87%E6%BB%A4%E5%99%A8)

#### 环绕轨迹

抛体将以发射者为中心执行持续的环绕运动。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/mN6aTimage.png)

- 环绕轨迹Key：相同环绕轨迹的关联键名，同一个Key值的不同抛体将会在同一个轨道上环绕运动
- 环绕中心偏移：基于以发射者为环绕中心位置进行偏移
- 环绕半径：环绕抛体环绕中心运动的圆周半径
- 初始相位偏移：环绕圆周运动开始时的初始相位偏移
- 中心点复位插值速度：当发射者在不断移动过程中，环绕中心点向其新位置平滑复位的速度；值越高，跟随越紧；值越低，跟随越松驰
- 旋转方向类型：抛体在环绕方向
    - 顺时针：环绕轨迹以顺时针进行环绕运动
    - 逆时针：环绕轨迹以逆时针进行环绕运动
- 抛体朝向类型：抛体在环绕过程中的朝向
    - 切线方向：抛体会朝向圆的切线方向
    - 逆切线方向：抛体会朝向圆的切线反方向
    - 圆周朝外：抛体会朝向圆心向外方向
    - 圆周朝内：抛体会朝向圆心向内方向
    - Z轴向上：抛体始终朝向Z轴向上方向
    - Z轴向下：抛体始终朝向Z轴向下方向
- 轨迹有效时间：环绕轨迹设定了一个控制有效时间，当抛体在轨迹上的时间超过此时限，将触发**逃逸**机制
- 抛体受控时间：环绕轨迹为单个抛体实例的最大控制时间，当单个抛体的环绕时间超过此时限，将触发**逃逸**机制
- 额外失效条件：环绕轨迹是否在其他情况下引起逃逸
   - 环绕Owner死亡时失效：发射者死亡时，所有环绕抛体**逃逸**
   - 环绕Owner销毁时失效：发射者被销毁时，所有环绕抛体**逃逸**
- 最大穿透次数：该环绕抛体可以穿透目标的最大次数
- 同单位碰撞触发间隔：同一个碰撞目标重复触发碰撞事件的间隔频率，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108) 

>逃逸：抛体立即脱离环绕运动，并保持逃逸瞬间的运动方向与速度，改为直线运动

重力系数和速度是否受曲线影响的配置详情跳转 [使用曲线](https://developer.gp.qq.com/wikieditor/#/catalog/20209?autoJump=%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF:~:text=%E7%8E%AF%E7%BB%95%E8%BD%A8%E8%BF%B9-,%E4%BD%BF%E7%94%A8%E6%9B%B2%E7%BA%BF,-%E7%A2%B0%E6%92%9E%E8%BF%87%E6%BB%A4%E5%99%A8)

#### 使用曲线

部分轨迹（抛物线、穿透、追踪）的速度、重力系数可以受到曲线的影响，可以实现类似于抛体先停住若干秒，然后飞行；或是抛体飞行一段时间，然后停住的效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/54Uksimage.png)

- 重力系数是否受曲线影响：使用曲线控制抛体飞行时的重力系数
- 速度是否受曲线影响：使用曲线控制抛体飞行时的速度
> 注意：重力曲线和速度曲线同时只能生效一个

以下图为例，使用曲线影响一个初始速度为500cm/s的抛体，引用一个初始为1，逐渐变为0的曲线，可使抛体速度逐渐衰减，第0秒抛体保持初始速度500cm/s，第2秒抛体速度为0cm/s。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pKFlvimage.png)

### 触发条件

于抛体生命周期内预置的特定事件。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/C0RXLimage.png)

- 发生碰撞调用：抛体的碰撞行为由其碰撞体组件的预设定义，严格遵循引擎的碰撞响应规则触发碰撞事件
- 停止时调用：抛体停止运动时将触发事件
- 销毁时调用：无论通过何种方式销毁抛体，均会正常触发销毁事件
- 碰撞地面时调用：抛体与地面发生碰撞时触发
- 碰撞单位时调用：抛体与单位目标碰撞时触发，单位目标特指玩家角色、怪物、载具等Pawn类对象
- 碰撞非单位时调用：抛体与非单位目标碰撞时触发，非单位目标指除玩家角色、怪物、载具等Pawn类对象外，其他的所有Actor对象
- 存活时持续调用：只要抛体还存活，就会以指定间隔时间持续触发
- 停止后持续调用：抛体停止运动，且还未销毁时，会以指定间隔时间持续触发
- 落地后持续调用：抛体运动最终停在地面后，且还未销毁时，会以指定间隔时间持续触发
- 弹射完成时调用：当抛体弹射次数用尽时触发事件，仅针对使用 ``目标弹射轨迹`` 的抛体生效
- 穿透完成时调用：当抛体穿透次数用尽时触发事件，仅针对使用 ``穿透轨迹`` 的抛体生效
- 环绕逃逸时调用：当抛体环绕次数用尽时触发事件，仅针对使用 ``环绕轨迹`` 的抛体生效
- 发射时调用：抛体被发射时触发

<br>

### 执行动作

当指定的事件触发时，执行的特定动作行为，支持为各动作独立添加过滤器与延迟时间。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pGjA6image.png)

- 过滤要求
	- 满足任意一个：满足任意一个过滤要求即触发
	- 全部满足：当所有的过滤条件均满足时才会触发
- 过滤器：应用 [通用过滤器](https://developer.gp.qq.com/wikieditor/#/catalog/20218) 决定是否触发动作，符合过滤要求（即判断结果为true）才允许触发 
- 动作延迟时间：每个动作都可以设定一个延迟执行时间，即从触发到真正开始执行所需的等待时间

> 执行端：客户端

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/z72Ajimage.png)

#### 自定义动作

当预设条件满足时，抛体会根据触发事件所关联的特定数据，动态执行由开发者自定义的行为逻辑。

创建一个继承自 ``Projectile Action Effect base`` 类的自定义抛体行为蓝图，并在该蓝图的Lua脚本中编写行为逻辑。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/aWn2Fimage.png)

在自定义动作脚本中，[``InitBP``](https://developer.gp.qq.com/api/#/searchContent/UProjectileActionEffectBase?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUProjectileActionEffectBase.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UProjectileActionEffectBase&autoJump=InitBP) 函数用于声明此动作的执行端，自定义行为逻辑均在 [``ApplyActionEffect``](https://developer.gp.qq.com/api/#/searchContent/UProjectileActionEffectBase?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUProjectileActionEffectBase.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UProjectileActionEffectBase&autoJump=ApplyActionEffect) 函数中实现。

以抛体命中时显示攻击者的头标为例：当抛体命中目标后，判断命中对象是否为角色，若为角色，则获取命中者的playerKey并显示对应头标，进而标识出攻击来源。此逻辑需设置在服务端运行，以确保命中效果能同步给所有客户端，使其他玩家也能看到。

**代码示例**

``` Lua
function TestLook:InitBP()
    -- ActionTriggerType 0 为只在服务端执行，1为只在客户单执行， 2为都会执行。默认为只在客户端执行
    self.ActionTriggerType = 0
end

function TestLook:ApplyActionEffect(TargetData)
    ugcprint(GetClassName((TargetData.HitResult.Actor):Get()))
    local actor = TargetData.HitResult.Actor:Get()
    local UIClass = UGCGameSystem.GetUGCResourcesFullPath(
        'Asset/Blueprint/showtime.showtime_C')
    if tostring(GetClassName(actor)) == tostring('UGCPlayerPawn') then
        UGCWidgetManagerSystem.AddObjectPositionUI(actor, UIClass,
            { X = 0, Y = 0, Z = 150 }, true, false, false, false)
    end
end
```

要应用已配置好的自定义动作，还需要将其配置到目标抛体的动作上：

### 脚本动态生成

UGCProjectileSystemV2静态库提供了 [``CreateProjectile``](https://developer-test.gp.qq.com/api/#/searchContent/UGCProjectileSystemV2?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCProjectileSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCProjectileSystemV2&autoJump=CreateProjectile) 函数支持动态生成并发射抛体。

示例代码：

```lua
local ProjectileClass = UGCObjectUtility.LoadClass(UGCGameSystem.GetUGCResourcesFullPath('Asset/Blueprint/Prefabs/Projectiles/Proj_Poison.Proj_Poison_C'))
local PlayerPawn = self:GetPlayerCharacterSafety()
local SelfPosition = PlayerPawn:K2_GetActorLocation()
local PlayerDirection = PlayerPawn:GetActorForwardVector()

local LaunchPosition = Vector.New(SelfPosition.X, SelfPosition.Y + 300, SelfPosition.Z + 100)
UGCProjectileSystemV2.CreateProjectile(ProjectileClass, self, LaunchPosition, PlayerDirection, 2000, 1, 20, ERestrictedDamageType.SkillDamage)
```

# 表格管理器

在游戏制作中，表格是存储和管理各类游戏数据的必备工具，表格管理器就是编辑器中专门用于筛选和处理表格的工具，它为开发者提供了一套统一的表格创建、浏览、编辑与数据流转解决方案。

<br>

## 表格类型

根据功能用途，表格管理器将表格划分为以下类型，各类型的具体功能与操作权限如下：

||自定义表格|功能表格|模板功能表格|内置表格|
|:-:|:-:|:-:|:-:|:-:|
|表格创建|√|√|||

``` lua
--UI控件的初始化函数
function MyUI:Construct()
	--给按键绑定点击事件回调
	self.ButtonViewAllData.OnClicked:Add(self.ButtonViewAllDataOnClicked, self);
	self.ButtonViewNewRowData.OnClicked:Add(self.ButtonViewNewRowDataOnClicked, self);
	--使用GetTableData获取表格内容
  self.MyTable = UGCGameSystem.GetTableData("Data/Table/Customized/myNewTable");
	--输出表格获取状况
  if self.MyTable ~= nil then
  	ugcprint("成功读取表格");
		--使用GetTableCount获取表格总行数
    local TotalCount = UGCGameSystem.GetTableCount("Data/Table/Customized/myNewTable");
    ugcprint(string.format("表格一共 %d 行数据", TotalCount));
  else
    ugcprint("未读取表格");
  end
end

function MyUI:ButtonViewAllDataOnClicked()
	ugcprint("查看所有数据");
	--遍历并输出表格的所有内容
  for k, v in pairs(self.MyTable) do
		ugcprint(string.format("RowName:%s|Data1:%s|Data2:%s|Data3:%s", tostring(k), tostring(v.Data1), tostring(v.Data2), tostring(v.Data3)));
	end
end

function MyUI:ButtonViewNewRowDataOnClicked()
	ugcprint("查看NewRow一行的数据");
	--方法1：使用GetTableDataByRowName获取对应表格的对应一行数据
	local NewRowData1 = UGCGameSystem.GetTableDataByRowName("Data/Table/Customized/myNewTable","NewRow");
	ugcprint(string.format("Data1:%s|Data2:%s|Data3:%s", tostring(NewRowData1.Data1), tostring(NewRowData1.Data2), tostring(NewRowData1.Data3)));
	--方法2：获取表格后直接利用索引获取对应一行数据
	local NewRowData2 = self.MyTable["NewRow"];
	ugcprint(string.format("Data1:%s|Data2:%s|Data3:%s", tostring(NewRowData2.Data1), tostring(NewRowData2.Data2), tostring(NewRowData2.Data3)));
end
```

## 注意事项

表格导入时必须满足以下条件，否则无法正常导入：
- 进行表格导入前，必须确保编辑器内与项目根目录中存在同名的表格文件，且双方均不可删除
- 列结构相同（列的数量/对应位置的名称/对应位置的数据类型）
- 根目录文件类型需要为.csv
- 根目录文件编码格式需要为utf-8，避免出现中文乱码
- 导出操作将自动包含其父文件夹结构（如：Asset\Data\Table）以维持此路径。导入文件时必须确保根目录的父文件夹结构与工程目录的完全一致。若文件被移动或文件夹结构改变导致路径不符，则导入失败
- 自定义结构体数据仅限表格管理器内修改，不支持导出在根目录修改

## 通用过滤器

> 文档路径: 通用功能 > 通用过滤器

# 通用过滤器

通用过滤器从给定的输入内容经过指定的过滤方式和筛选条件，输出满足要求的结果集，适合作为属性组配置应用在需要筛选结果的功能蓝图中。

<br>

## 过滤条件

过滤器输出结果的必要条件，结果依应用场景的语境不同而具备不同的含义，通常条件包含两种：

- 全部满足：所有的过滤器条件全部满足才能作为输出结果
- 满足任意一个：满足任意一个过滤器条件即可作为输出结果之一

<br>

## 过滤类型

### Actor类过滤器

按指定的Actor类进行目标过滤。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/CE5DGimage.png)

- Explicit Class：当启用该选项时，系统将仅检测当前指定的Actor类；否则，检测范围会自动包含Actor类的所有派生子类
- Included Actor Classes：指定的Actor类列表
- Invert Filter：是否对过滤结果取反

---

### 阵营过滤器

基于阵营对目标进行过滤。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/9gZOOimage.png)

- Filter Camp：仅选择该阵营的目标
- Invert Filter：是否对过滤结果取反

---

### 非队友过滤器

过滤目标是否排除队友。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/IAoNrimage.png)

- Invert Filter：是否对过滤结果取反，即允许选择队友

---

### 可见性过滤器

基于射线检测的方法对Actor对象的可见性进行目标过滤。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/StxkNimage.png)

- Ignore Actor Classes：在射线检测时忽略的特定Actor类
- Ignore Actor Soft Classes：射线可见性检测忽略的Actor类（软引用），通常建议配置此参数
- Trace Eyes：若勾选，则射线从眼球为起点发射；否则，从Pawn位置为起点发射
- Block Channel: 会阻挡可见性的通道类型
- Invert Filter：是否对过滤结果取反，即选择不可见的目标

---

### 概率过滤器

每次检测时，系统将根据预设的概率，随机返回 True 或 False 。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Mb3uEimage.png)

- 生效概率：该过滤器每次检测，有多大的概率为True，支持 [属性绑定](https://developer.gp.qq.com/wikieditor/#/catalog/20108)
- Invert Filter：是否对随机结果取反

> 属性绑定数据类型：float

---

### 自定义条件过滤器

允许绑定一个返回布尔值的外部函数，执行过滤时，将调用此函数，并将其返回值直接作为过滤结果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Nyq32image.png)

- TriggerFunction：该过滤器绑定的自定义函数
- Invert Filter：是否对返回值的结果取反

---

### 实体过滤器

根据预设的实体类型列表进行检测，若目标的实体类型匹配列表中任一类型，则返回true。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/NkSOcimage.png)

- 包含的实体类型：需要配置实体类型对应的不同 [GameplayTag](https://developer.gp.qq.com/wikieditor/?timeStamp=1725589224096#/catalog/20102)
- Invert Filter：是否对匹配的结果取反

# Tween 动画使用指南

**功能名称**

Tween 动画系统提供了一套完整的差值平滑动画能力，支持位置、旋转、颜色等多种数值类型的平滑过渡，适用于 UI 动效、对象移动、场景变换等多种场景。

## 功能概述

Tween 动画系统是一套基于时间差值的动画框架，通过在指定时间内将数值从起始值平滑过渡到目标值，配合多种缓动算法实现丰富的动画效果。

### 适用场景

Tween 动画系统特别适合以下场景：
- **UI 动效**：按钮缩放、淡入淡出、颜色变化等界面动画
- **对象移动**：Actor 位置移动、旋转动画
- **场景变换**：摄像机平滑移动、场景元素过渡
- **数值动画**：进度条显示、数值计数动画
- **视觉反馈**：击中特效、状态变化反馈

**注意，Tween动画只在客户端生效，相关的接口也只能在客户端调用。**

### 核心优势

相较于传统的动画实现方式，Tween 动画系统具有以下优势：
- **简单易用**：只需指定起始值、目标值、持续时间和缓动类型即可创建动画
- **灵活控制**：支持暂停、恢复、停止、循环、往返等多种控制方式
- **链式调用**：支持多个动画串联执行，实现复杂的动画序列
- **低学习成本**：基于 Lua API 设计，开发者无需深入了解底层实现

## 功能详解

### 前置依赖

无前置依赖条件。Tween 动画系统作为 UGC 编辑器的内置功能提供，开发者可以直接在 Lua 脚本中调用相关 API。

### 整体简介

#### 功能背景

在游戏开发中，经常需要实现各种平滑过渡动画效果，例如 UI 元素的淡入淡出、按钮的缩放反馈、场景中物体的移动等。传统的实现方式可能需要：
- 手动编写复杂的插值算法
- 在 Tick 中逐帧更新数值
- 维护动画状态和生命周期

Tween 动画系统提供了一种更简洁的解决方案：
- 通过 API 直接创建动画，自动处理插值计算
- 内置多种缓动算法，只需选择类型即可
- 支持动画控制（暂停、恢复、停止、循环）
- 提供回调机制，在动画完成时执行后续逻辑

#### 工作原理

Tween 动画系统的核心原理是将数值从起始值平滑过渡到目标值，通过缓动函数调整动画的节奏和加速度。

**基本流程：**

1. **定义数值范围**：指定起始值和目标值
2. **设置动画参数**：持续时间和缓动类型
3. **创建动画**：通过 API 创建 Tween 动画
4. **自动执行**：系统在每一帧自动计算当前值
5. **应用数值**：通过回调函数将当前值应用到目标对象

### 详细配置

#### 支持的动画类型

Tween 动画系统支持以下数值类型的动画：

| 动画类型 | API 函数 | 说明 | 适用场景 |
|---------|---------|------|---------|
| **Actor 位置** | `UGCTweenSystem.TweenActorLocation` | Actor 物体位置移动 | 角色移动、道具动画 |
| **Actor 旋转** | `UGCTweenSystem.TweenActorRotation` | Actor 物体旋转动画 | 角色转身、道具旋转 |
| **浮点数值** | `UGCTweenSystem.TweenFloatValue` | 浮点数值变化 | 进度条、数值显示 |
| **向量数值** | `UGCTweenSystem.TweenVectorValue` | 3D 向量变化 | UI 元素移动、位置调整 |
| **旋转器数值** | `UGCTweenSystem.TweenRotatorValue` | 旋转器变化 | 角度控制、方向变化 |
| **颜色数值** | `UGCTweenSystem.TweenColorValue` | 颜色渐变 | UI 颜色变化、特效颜色 |

#### 缓动类型（Easing Type）

缓动类型决定了动画的加速度和节奏，系统提供了 33 种缓动算法，对应 Tween.js 的缓动类型：

**基础缓动：**

| 缓动类型 | 效果 | 使用场景 |
|---------|------|---------|
| `Linear` | 线性匀速 | 持续性动画、机械运动 |
| `QuadIn` | 二次加速 | 开始缓慢、逐渐加速 |
| `QuadOut` | 二次减速 | 开始快速、逐渐减速 |
| `QuadInOut` | 二次往返 | 平滑加速减速 |
| `CubicIn/Out/InOut` | 三次缓动 | 平滑过渡效果 |
| `QuartIn/Out/InOut` | 四次缓动 | 较强的加速/减速效果 |

**特殊缓动：**

| 缓动类型 | 效果 | 使用场景 |
|---------|------|---------|
| `SineIn/Out/InOut` | 正弦缓动 | 自然平滑的过渡 |
| `ExpoIn/Out/InOut` | 指数缓动 | 快速加速或减速 |
| `CircIn/Out/InOut` | 圆形缓动 | 圆周运动相关 |
| `ElasticIn/Out/InOut` | 弹性缓动 | 弹簧效果、回弹动画 |
| `BackIn/Out/InOut` | 超出回退 | 超出目标后返回 |
| `BounceIn/Out/InOut` | 弹跳缓动 | 弹跳效果、落地动画 |

**完整缓动类型列表：**
```lua

CubicIn, CubicOut, CubicInOut,
QuartIn, QuartOut, QuartInOut,
QuintIn, QuintOut, QuintInOut,
SineIn, SineOut, SineInOut,
ExpoIn, ExpoOut, ExpoInOut,
CircIn, CircOut, CircInOut,
ElasticIn, ElasticOut, ElasticInOut,
BackIn, BackOut, BackInOut,
BounceIn, BounceOut, BounceInOut
```

#### 动画参数配置

Tween 动画支持以下配置参数，通过 `UGCTweenSystem.MakeConfig()` 创建：

```lua
local Config = UGCTweenSystem.MakeConfig(0, 1, false, 0)
```

**参数说明：**

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `Delay` | float | 初始延迟时间（秒） | 0 |
| `RepeatCount` | int32 | 重复次数（-1=无限，0=不重复） | 1 |
| `bYoyo` | bool | 是否往返（A→B→A） | false |
| `RepeatDelay` | float | 每次重复前的等待时间（秒） | 0 |

**参数详细说明：**

- **延迟**：动画执行前的等待时间。例如设置 `Delay = 2.0`，则动画会在 2 秒后才开始执行。
- **循环次数**：
  - `RepeatCount = 0`：只播放一次
  - `RepeatCount = 1`：播放完成后再播放一次
  - `RepeatCount = -1`：无限循环
  - `RepeatCount = N`：总共播放 N+1 次（1 次初始 + N 次重复）
- **往返动画**：开启后，动画会先从起始值到目标值，再从目标值回到起始值。例如设置 `bYoyo = true`，则动画路径为 A→B→A。
- **循环延迟**：每次重复前的等待时间。只在重复次数 > 0 时生效。

### 运行时使用方式

#### 获取 Tween 子系统

通过 `UGCTweenSystem.GetTweenSubsystem()` 获取 Tween 子系统实例，可用于动画配置：

```lua
local TweenSubsystem = UGCTweenSystem.GetTweenSubsystem()
```

#### 创建回调函数

Tween 动画通过回调函数来应用当前值到目标对象。回调函数会在动画每一帧执行，接收当前值作为参数。

**创建回调函数：**

```lua
local Callback = function(Object, Value)
    -- Object 是系统自动传入的占位符（第一个参数）
    -- Value 是当前动画的数值（第二个参数）
    -- 在这里将 Value 应用到目标对象
    self.TargetWidget:SetColorAndOpacity(Value)
end
```

#### 创建 Tween 动画

以下是创建各种类型 Tween 动画的示例：

##### 颜色动画示例

```lua
function TweenColor:Construct()
    -- 定义起始颜色和目标颜色
    local StartColor = KismetMathLibrary.MakeColor(0, 1, 0, 1)  -- 绿色
    local EndColor = KismetMathLibrary.MakeColor(1, 0, 0, 1)    -- 红色

    local Callback = function(Object, Value)
        -- Object 是系统自动传入的占位符
        self.TweenTarget:SetColorAndOpacity(Value)
    end

    -- 配置动画参数
    local Config = UGCTweenSystem.MakeConfig(0, -1, false, 0)

    -- 创建颜色动画（1秒持续时间，使用 Linear 缓动）
    self.TweenHandler = UGCTweenSystem.TweenColorValue(
        StartColor,
        EndColor,
        1.0,
        self.TweenType,        -- 缓动类型
        Callback,
        Config
    )

    -- 暂停动画
    UGCTweenSystem.PauseTween(self.TweenHandler)
end
```

#### 控制动画

Tween 动画提供了多种控制方式：

##### 暂停/恢复动画

```lua
function PauseCheckBox_OnCheckStateChanged(bIsChecked)
    if not self.TweenHandler then
        return
    end

    if bIsChecked then
        -- 暂停动画
```

##### 修改动画配置（往返、循环）

```lua
function YoyoCheckBox_OnCheckStateChanged(bIsChecked)

        return
    end

    -- 动态修改动画配置
    UGCTweenSystem.ConfigureTween(
        self.TweenHandler,
        0,               -- Delay
        -1,              -- RepeatCount（-1 表示无限循环）
        bIsChecked,      -- bYoyo
        0                -- RepeatDelay
    )
end
```

##### 停止动画

```lua
function StopAnimation()
```

```lua
function ChainAnimations()
```

```lua
    local TweenHandler = UGCTweenSystem.TweenFloatValue(
        StartProgress,
        EndProgress,
        5.0,
        EEasingType.Linear,
        Callback,
        Config
    )

    -- 绑定完成回调
    UGCTweenSystem.BindCompletedDelegate(TweenHandler, function()
        print("加载完成！")
        ProgressText:SetText("加载完成: 100%")
    end)
end
```

### 场景五：组合动画序列

先淡入 UI，再移动到指定位置，最后缩放放大：

```lua
function CombinedAnimationSequence()
    local UIWidget = self.AnimatedWidget

    -- 步骤1：淡入
    local FadeCallback = function(Object, Value)
        -- Object 是系统自动传入的占位符
        UIWidget:SetRenderOpacity(Value)
    end

    local FadeConfig = UGCTweenSystem.MakeConfig(0, 0, false, 0)

    local FadeHandler = UGCTweenSystem.TweenFloatValue(
        0.0,
        1.0,
        1.0,
        EEasingType.EaseInQuad,
        FadeCallback,
        FadeConfig
    )

    -- 步骤2：移动
    local MoveCallback = function(Object, Value)
        -- Object 是系统自动传入的占位符
        local Slot = UGCWidgetManagerSystem.SlotAsCanvasSlot(UIWidget)
        Slot:SetPosition(UGCMathUtility.MakeVector2D(Value.X, Value.Y))
    end

    local MoveConfig = UGCTweenSystem.MakeConfig(0, 0, false, 0)

    local MoveHandler = UGCTweenSystem.TweenVectorValue(
        UGCMathUtility.MakeVector(0, 0, 0),
        UGCMathUtility.MakeVector(200, 100, 0),
        1.0,
        EEasingType.EaseOutQuad,
        MoveCallback,
        MoveConfig
    )

    -- 步骤3：缩放
    local ScaleCallback = function(Object, Value)
        -- Object 是系统自动传入的占位符
        UIWidget:SetRenderScale(UGCMathUtility.MakeVector2D(Value, Value))
    end

    local ScaleConfig = UGCTweenSystem.MakeConfig(0, 0, false, 0)

    local ScaleHandler = UGCTweenSystem.TweenFloatValue(
        1.0,
        1.5,
        0.5,
        EEasingType.EaseOutQuad,
        ScaleCallback,
        ScaleConfig
    )

    -- 链接动画序列
    UGCTweenSystem.ChainTween(FadeHandler, MoveHandler)
    UGCTweenSystem.ChainTween(MoveHandler, ScaleHandler)
end
```

## 使用注意事项

### 基本使用规则

- **正确配置 Config**：通过 `UGCTweenSystem.MakeConfig()` 创建配置
- **回调函数绑定**：使用 `function(Object, Value)` 创建回调函数，其中 Object 是系统自动传入的占位符，Value 是当前的数值，在回调中处理数值应用
- **保存 TweenHandler**：将创建的 `FTweenHandle` 保存到变量中，以便后续控制（暂停、恢复、停止）

### 循环和往返动画

- **往返动画**：设置 `bYoyo = true` 时，动画会从起始值到目标值再回到起始值
- **循环次数**：
  - `RepeatCount = 0`：只播放一次（无往返则 A→B，有往返则 A→B→A）
  - `RepeatCount = 1`：播放完成后再播放一次
  - `RepeatCount = -1`：无限循环
  - `RepeatCount = N`：总共播放 N+1 次
- **循环延迟**：只在重复次数 > 0 时生效，每次重复前的等待时间
- **往返延迟**：往返动画在返回阶段的延迟时间

### 性能优化

- **避免频繁创建动画**：重复使用的动画可以复用 TweenHandler
- **及时停止无用动画**：动画完成后或不再需要时，调用 `KillTween` 停止动画
- **合理设置循环次数**：无限循环的动画会持续消耗资源，谨慎使用
- **批量操作时使用链式动画**：避免同时创建大量动画导致性能下降

### 回调函数使用

- **回调频率**：回调函数会在每一帧执行，避免在回调中执行耗时操作
- **参数说明**：回调函数的第一个参数 `Object` 是系统自动传入的占位符，第二个参数才是当前的数值 `Value`
- **数值应用**：回调中的 `Value` 是当前动画的数值，直接应用到目标对象
- **避免死循环**：不要在回调中创建新的 Tween 动画，否则可能导致无限循环

### 动画冲突

- **同一对象多个动画**：避免对同一属性同时执行多个动画，会产生冲突
- **使用链式动画**：需要对同一对象执行多个动画时，使用 `ChainTween` 串联执行
- **停止动画时注意**：调用 `KillTween` 会停止动画并将对象恢复到起始值

### 为什么动画没有执行？

常见原因：
1. **未暂停后恢复**：创建动画后默认会自动执行，如果需要延迟执行，应先调用 `PauseTween` 暂停，然后在需要时调用 `ResumeTween` 恢复
2. **回调函数未正确绑定**：检查回调函数是否为有效的 Lua 函数，并正确接收参数值
3. **对象不存在**：确保目标对象（Widget、Actor）在动画执行时存在

### 如何让动画延迟执行？

设置 `Config.Delay` 参数：

```lua
local Config = UGCTweenSystem.MakeConfig(2.0, 0, false, 0)  -- 延迟 2 秒后执行
```

### 如何实现动画循环？

设置 `Config.RepeatCount` 参数：

```lua
-- 无限循环
local Config = UGCTweenSystem.MakeConfig(0, -1, false, 0)

-- 播放 3 次（1 次初始 + 2 次重复）
local Config = UGCTweenSystem.MakeConfig(0, 2, false, 0)
```

### 如何实现往返动画（A→B→A）？

设置 `Config.bYoyo = true`：

```lua
local Config = UGCTweenSystem.MakeConfig(0, -1, true, 0)  -- 往返循环
```

### 如何选择合适的缓动类型？

根据动画效果选择：
- **Linear**：匀速动画，持续性运动
- **EaseOut**：减速动画，自然停止效果
- **EaseIn**：加速动画，从静止开始
- **EaseInOut**：平滑加速减速，过渡自然
- **Elastic**：弹性动画，回弹效果
- **Bounce**：弹跳动画，落地效果

建议先在 `Template_PlaygroundHub` 工程的 Tween 动画演示页面中查看各种缓动类型的效果。

### 如何停止动画？

根据需求选择：
- **KillTween**：停止动画并恢复到起始值
- **PauseTween**：暂停动画，保持当前值
- **等待动画完成**：使用 `BindCompletedDelegate` 绑定完成回调

### 如何创建复杂动画序列？

使用链式动画 `ChainTween`：

```lua
-- 创建三个动画
local Handler1 = UGCTweenSystem.TweenFloatValue(...)
local Handler2 = UGCTweenSystem.TweenFloatValue(...)
local Handler3 = UGCTweenSystem.TweenFloatValue(...)

-- 链接动画序列
UGCTweenSystem.ChainTween(Handler1, Handler2)
UGCTweenSystem.ChainTween(Handler2, Handler3)
```

### 循环动画会在什么时候触发完成回调？

- **RepeatCount = 0 或 RepeatCount > 0 且 bYoyo = false**：动画完成时触发
- **RepeatCount > 0 且 bYoyo = true**：每次往返完成时触发（包括最后一次）
- **RepeatCount = -1**：不会触发完成回调

### 如何在动画执行过程中修改参数？

使用 `UGCTweenSystem.ConfigureTween`：

```lua
UGCTweenSystem.ConfigureTween(
    TweenHandler,
    0,          -- Delay
    -1,         -- RepeatCount
    true,       -- bYoyo
    0           -- RepeatDelay
)
```

### 同一个对象可以同时执行多个动画吗？

技术上可以，但建议避免：
- **不同属性**：可以同时执行（如位置和旋转）
- **同一属性**：会产生冲突，导致动画效果混乱
- **解决方案**：使用链式动画串联执行，或停止当前动画后再创建新动画
