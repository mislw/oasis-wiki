# 工程概述
## 项目简介

1、团队竞技工程（Template\_CTCompetition2）是一款支持最高4v4的pvp晋级模板
2、创建该模板后，可不进行任何修改就能直接完成4v4的团队竞技的玩法

### 胜利条件

1. **分数获胜**：任意队伍达到设定的胜利分数（默认 50 分）
2. **时间获胜**：战斗时间耗尽后分数高的队伍获胜
3. **对手离场**：对手全部离开或某队人数为 0

##  核心系统概览

### 1. 游戏模式系统（UGCGameMode）

负责游戏核心逻辑控制，包括：

* 游戏流程状态管理（准备→战斗→结算）
* 玩家登录/退出处理
* 队伍分配与切换
* 击杀事件处理
* 胜负判定
* 复活逻辑

**关键参数**：

* CFG\_MaxTeamMember：每队最大人数（1-4）
* CFG\_PrepareTime：准备时间（10-60秒）
* CFG\_BattleTime：战斗时间（90-600秒）
* CFG\_WinScore：胜利分数（10-200）
* CFG\_RespawnInterval：复活等待时间（0.5-10秒）

### 2. 游戏状态系统（UGCGameState）

管理同步的游戏状态数据，包括：

* 当前游戏阶段
* 队伍数据（玩家列表、UID）
* 战绩数据（击杀/死亡/助攻）
* 队伍积分
* UI 显示控制

**核心功能**：

* 数据同步（服务器→客户端）
* 队伍信息更新
* 战绩实时计算
* UI 触发显示

### 3. 玩家控制器系统（UGCPlayerController）

处理玩家输入和交互，包括：

* 队伍切换请求
* 装备配置修改
* UI 加载控制
* RPC 服务端处理

**可用 RPC**：

* ServerChangeTeam：切换队伍
* ServerChangeWeaponList：修改武器配置
* ServerChangeWeapons：设置完整武器列表

### 4. 玩家状态系统（UGCPlayerState）

存储玩家个人数据，包括：

* 武器配置列表
* 伤害来源列表（助攻统计）
* 数据同步标记

### 5. 事件系统（UGCEventSystem）

实现事件订阅/发布机制，用于：

* 游戏状态变更通知
* 队伍数据更新通知
* 战绩数据更新通知
* UI 事件触发

**客户端事件类型**：

* GameStatusChanged：游戏状态变更
* TeamDataChanged：队伍数据变更
* PlayerScoreChanged：玩家战绩变更
* TeamScoreChanged：队伍积分变更
* GameEndEvent：游戏结束

### 6. 装备系统

支持玩家自定义装备配置，包括：

* 武器选择
* 配件选择（瞄具、枪口、弹夹等）
* 配件数量设置
* 装备范围限制

**配置结构**：

```
WeaponList[1] = {
    ItemID = 武器ID,
    Count = 武器数量,
    SubItemList = {
        [1] = {ItemID = 配件ID, Count = 配件数量},
        [2] = {ItemID = 配件ID, Count = 配件数量},
        ...
    }
}
```

##  常见问题

**Q: 如何修改队伍数量？**

A: 当前仅支持两队竞技（红蓝），如需多队需要修改 CTCompetitionEnum.lua 中的 ETeamType 枚举和 UGCGameState.lua 中的队伍逻辑。


**Q: 装备配置不生效？**

A: 检查 CFG\_IsEnablePlayerWeaponConfig 是否为 true，以及装备 ID 是否在允许范围内。

##  枚举定义

### CTCompetitioClientEvent.lua

**文件路径**：Script/Common/CTCompetitioClientEvent.lua

**作用**：定义客户端可接收的游戏事件

### 事件列表

|事件ID|事件名称|说明|
|-----|-----|-----|
|101|GameStatusChanged|游戏状态变更|
|102|NextStatusTimeStampChanged|阶段结束时间变更|
|103|TeamDataChanged|队伍数据变更|
|104|TeamUIDChanged|队伍 UID 变更|
|105|PlayerScoreChanged|玩家战绩变更|
|106|TeamScoreChanged|队伍积分变更|
|107|GameEndEvent|游戏结束事件|
|108|ShowCommonTipsUI|显示提示消息|
|109|EquipSelectRangeChanged|装备选择范围变更|

**原代码路径**：Script/Common/CTCompetitioClientEvent.lua:4-53

### 事件详解

#### GameStatusChanged

**说明**：游戏状态变更事件

**参数**：

* CurrentGameStatus：新的游戏状态（EGameStatus 枚举）

**触发时机**：

* 进入准备阶段
* 进入战斗阶段
* 进入结算阶段

**使用方式演示**：

```
-- 在 UI 脚本中订阅事件
function MyWidget:OnWidgetShow()
    UGCEventSystem:RegisterEvent(
        CTCompetitioClientEvent.GameStatusChanged, 
        self, 
        self.OnGameStatusChanged
    )
end

function MyWidget:OnGameStatusChanged(NewStatus)
    if NewStatus == EGameStatus.Prepare then
        -- 显示准备阶段 UI
        self:ShowPrepareUI()
    elseif NewStatus == EGameStatus.Battle then
        -- 显示战斗阶段 UI
        self:ShowBattleUI()
    elseif NewStatus == EGameStatus.Result then
        -- 显示结算界面
        self:ShowResultUI()
    end
end
```

#### NextStatusTimeStampChanged

**说明**：阶段结束时间变更事件

**参数**：

* NextStatusTimeStamp：下一阶段的结束时间（世界时间戳）

```
function MyWidget:OnNextStatusTimeStampChanged(TimeStamp)

        math.floor(RemainingTime % 60)))
end
```

#### GameEndEvent

**说明**：游戏结束事件

**参数**：

```
BattleResult = {
    WinTeam = 1,  -- 0=平局, 1=红队, 2=蓝队
    TeamScore = {
        [1] = 50,
        [2] = 30
    },
    PlayerScore = {
        [PlayerKey1] = {
            KillCount = 5,
            DeathCount = 2,
            AssistCount = 3,
            TeamID = 1,
            UID = "10001"
        },
        ...
    }
}
```

**使用方式演示**：

```
function MyWidget:OnGameEndEvent(BattleResult)
    -- 判定获胜队伍
    if BattleResult.WinTeam == ETeamType.Red then
        self.WinnerText:SetText("红队获胜！")
        self.WinnerIcon:SetColor(FLinearColor(1.0, 0.2, 0.2, 1.0))  -- 红色
    elseif BattleResult.WinTeam == ETeamType.Blue then
        self.WinnerText:SetText("蓝队获胜！")
        self.WinnerIcon:SetColor(FLinearColor(0.2, 0.5, 1.0, 1.0))  -- 蓝色
    else
        self.WinnerText:SetText("平局！")
        self.WinnerIcon:SetColor(FLinearColor(0.8, 0.8, 0.8, 1.0))  -- 灰色
    end
    
    -- 显示最终战绩
    self:ShowFinalScoreboard(BattleResult.PlayerScore)
    
    -- 判定 MVP（击杀数最多的玩家）
    local MVPPlayerKey = nil
    local MaxKillCount = 0
    
    for PlayerKey, ScoreData in pairs(BattleResult.PlayerScore) do
        if ScoreData.KillCount > MaxKillCount then
            MaxKillCount = ScoreData.KillCount
            MVPPlayerKey = PlayerKey
        end
    end
    
    if MVPPlayerKey then
        local PS = UGCGameSystem.GetPlayerStateByPlayerKey(MVPPlayerKey)
        if PS then
            self:MVPPlayerText:SetText("MVP: " .. PS.PlayerName)
        end
    end
end
```

### 1. 添加自定义事件

**步骤**：

1. 在 CTCompetitioClientEvent.lua 中定义新事件 ID
2. 在 UGCGameState.lua 中发布事件
3. 在 UI 脚本中订阅事件

**示例：添加击杀通知事件**：

```
-- 1. 在 CTCompetitioClientEvent.lua 中添加
CTCompetitioClientEvent.KillNotification = 110

-- 参数格式
--[[
params:
  ├─ KillerName: "张三"
  ├─ VictimName: "李四"
  ├─ WeaponName: "M416"
  └─ IsHeadshot: true
--]]

-- 2. 在 UGCGameState.lua 中发布事件
function UGCGameState:UpdateScore(Killer, VictimPlayer)
    -- 原有战绩更新逻辑...
    
    -- 发布击杀通知事件
    if Killer and VictimPlayer then
        local KillerPS = UGCGameSystem.GetPlayerStateByPlayerKey(Killer.PlayerKey)
        local VictimPS = UGCGameSystem.GetPlayerStateByPlayerKey(VictimPlayer.PlayerKey)
        
        if KillerPS and VictimPS then
            local KillData = {
                KillerName = KillerPS.PlayerName,
                VictimName = VictimPS.PlayerName,
                WeaponName = "M416",  -- 可根据实际情况获取
                IsHeadshot = false
            }
            
            UGCEventSystem:SendEvent(CTCompetitioClientEvent.KillNotification, KillData)
        end
    end
end

-- 3. 在 UI 脚本中订阅事件
function MyWidget:OnWidgetShow()
    UGCEventSystem:RegisterEvent(
        CTCompetitioClientEvent.KillNotification, 
        self, 
        self.OnKillNotification
    )
end

function MyWidget:OnKillNotification(KillData)
    local Message = string.format("%s 击杀了 %s", KillData.KillerName, KillData.VictimName)
    if KillData.IsHeadshot then
        Message = Message .. " [爆头！]"
    end
    
    self:AddKillMessage(Message)
end
```

##  游戏阶段管理

### SetStatus(InStatus, InDuration)

**功能**：切换游戏阶段并设置持续时间

**参数**：

* InStatus：目标阶段（EGameStatus 枚举）
* InDuration：持续时间（秒）

**处理逻辑**：

1. 更新当前阶段
2. 计算阶段结束时间
3. 根据阶段设置玩家无敌状态
4. 结算阶段准备并下发结算数据
5. 同步数据到客户端

```
function UGCGameState:SetStatus(InStatus, InDuration)
    self.CurrentGameStatus = InStatus
    self.NextStatusTimeStamp = self:GetWorldTime() + InDuration
    
    ugcprint("UGCGameMode:SetStatus InStatus="..tostring(InStatus).." InDuration="..tostring(InDuration).." NextStatusTimeStamp="..tostring(self.NextStatusTimeStamp))
    
    -- 根据阶段设置无敌状态
    if self.CurrentGameStatus == EGameStatus.Prepare then
        local AllPawn = UGCGameSystem.GetAllPlayerPawn()
        for _, Pawn in ipairs(AllPawn) do
            UGCPlayerPawnSystem.SetIsInvincible(Pawn, true)
        end
    else
        local AllPawn = UGCGameSystem.GetAllPlayerPawn()
        for _, Pawn in ipairs(AllPawn) do
            UGCPlayerPawnSystem.SetIsInvincible(Pawn, false)
        end
    end
    
    -- 结算阶段准备
    if self.CurrentGameStatus == EGameStatus.Result then
        -- 判定胜利队伍
        if self.TeamScore[ETeamType.Red] == self.TeamScore[ETeamType.Blue] then
            self.BattleResult.WinTeam = 0
        elseif self.TeamScore[ETeamType.Red] > self.TeamScore[ETeamType.Blue] then
            self.BattleResult.WinTeam = ETeamType.Red
        else
            self.BattleResult.WinTeam = ETeamType.Blue
        end

        log_tree("UGCGameMode:Send BattleResult", self.BattleResult)

    UnrealNetwork.RepLazyProperty(self, "NextStatusTimeStamp")
```

### UpdateScore(Killer, VictimPlayer)

**功能**：更新击杀/死亡/助攻数据（服务器端）

**计分规则**：

* **击杀**：击杀者 KillCount + 1
* **死亡**：被击杀者 DeathCount + 1
* **助攻**：造成伤害的非击杀者 AssistCount + 1

```
function UGCGameState:UpdateScore(Killer, VictimPlayer)
    local PlayerScore = self.PlayerScore
    
    -- 击杀者加分（必须不同队）
    if Killer and VictimPlayer and Killer.TeamID ~= VictimPlayer.TeamID then
        self:TryInitPlayerScore(Killer)
        PlayerScore[Killer.PlayerKey].KillCount = PlayerScore[Killer.PlayerKey].KillCount + 1
    end
    
    -- 被击杀者记录死亡
    if VictimPlayer then
        self:TryInitPlayerScore(VictimPlayer)
        PlayerScore[VictimPlayer.PlayerKey].DeathCount = PlayerScore[VictimPlayer.PlayerKey].DeathCount + 1
        
        -- 助攻判定（造成伤害的其他玩家）
        if VictimPlayer.PlayerState and next(VictimPlayer.PlayerState.DamagePlayerKeys) ~= nil then
            for k,v in pairs(VictimPlayer.PlayerState.DamagePlayerKeys) do
                if k ~= Killer.PlayerKey and k ~= VictimPlayer.PlayerKey then
                    local AssistPC = UGCGameSystem.GetPlayerControllerByPlayerKey(k)
                    if AssistPC then
                        self:TryInitPlayerScore(AssistPC)
                        PlayerScore[k].AssistCount = PlayerScore[k].AssistCount + 1
                    end
                    break
                end
            end
        end
    end
    
    log_tree("UGCGameState:UpdateScore", PlayerScore)
    
    -- 同步数据
    UnrealNetwork.RepLazyProperty(self, "PlayerScore")
    
    -- 更新队伍积分
    self:UpdateTeamScore()
end
```

### ServerChangeTeam(NewTeamID, bForce)

**RPC 函数**：客户端调用，服务器执行

**参数**：

* NewTeamID：目标队伍 ID（Red=1, Blue=2）
* bForce：是否强制切换（忽略阶段限制）

**功能**：

1. 获取游戏模式
2. 检查队伍 ID 有效性
3. 检查是否在准备阶段或强制切换
4. 调用游戏模式的队伍切换逻辑

**调用条件**：

* 目标队伍为红队或蓝队
* 当前在准备阶段 或 bForce 为 true

```
function UGCPlayerController:ServerChangeTeam(NewTeamID, bForce)
    local GameMode = UGCGameSystem.GetGameMode()
    
    ugcprint("UGCPlayerController:ServerChangeTeam "..tostring(self.PlayerName)
        .." TeamID = "..tostring(self.TeamID).." NewTeamID = "..tostring(NewTeamID) 
        .. "GameMode.GameState.CurrentGameStatus="..tostring(GameMode.GameState.CurrentGameStatus))
    
    if GameMode then
        -- 检查是否可以切换队伍
        if NewTeamID ~= self.TeamID and 
           (GameMode.GameState.CurrentGameStatus == EGameStatus.Prepare or bForce) then
            ugcprint("UGCPlayerController:ServerChangeTeam ChangeTeamFinish.......1111")
            GameMode:PlayerChangeTeam(self.PlayerKey, NewTeamID)
        end
    else
        ugcprint("UGCPlayerController:ServerChangeTeam ChangeTeamFinish")
    end
end
```

### ServerChangeWeaponList(Index, SubIndex, NewID, NewCount)

**RPC 函数**：客户端调用，服务器执行

**参数**：

* Index：武器槽位索引（≥1）
* SubIndex：配件槽位索引（0=主武器，≥1=配件）
* NewID：新的物品 ID（0=清空）
* NewCount：新的物品数量（可选）

**功能**：

1. 检查是否在准备阶段
2. 记录最后编辑的武器槽位
3. 根据索引类型更新主武器或配件
4. 如果索引超出当前列表长度，扩展列表
5. 同步武器列表到玩家状态

**调用条件**：

* 当前游戏阶段为准备阶段
* Index 为有效正整数

**参数说明**：

|参数|说明|示例|
|-----|-----|-----|
Index|武器槽位（1=第1把武器，2=第2把武器...）|1|
SubIndex|配件槽位（0=主武器，1=配件1，2=配件2...）|0|
NewID|新的物品 ID（0=清空，-1=占位）|1001|
NewCount|新的物品数量（可选）|1|

**逻辑图解**：

```
WeaponList[1] = {
    ItemID = 1001,        ← SubIndex=0 修改此处
    Count = 1,
    SubItemList = {
        [1] = {ItemID = 2001, Count = 1},  ← SubIndex=1 修改此处
        [2] = {ItemID = 2002, Count = 1}   ← SubIndex=2 修改此处
    }
}
```

```
function UGCPlayerController:ServerChangeWeaponList(Index, SubIndex, NewID, NewCount)
    local GameMode = UGCGameSystem.GetGameMode()
    
    if GameMode and GameMode.GameState.CurrentGameStatus == EGameStatus.Prepare then
        local WeaponList = self.PlayerState.WeaponList
        local WeaponListNum = #WeaponList
        
        self.LastWeaponListIndex = Index
        
        -- 默认 ItemID 为 0（清空）
        if NewID == nil then
            NewID = 0
        end
        
        -- 修改现有武器
        if Index > 0 and Index <= WeaponListNum then
            ugcprint("WeaponListNum "..tostring(WeaponListNum))
            local WeaponConfig = WeaponList[Index]
            local SubItemListNum = #WeaponConfig.SubItemList
            
            -- 修改配件
            if SubIndex > 0 then
                ugcprint("SubItemListNum "..tostring(SubItemListNum))
                
                if SubIndex <= SubItemListNum then
                    -- 修改现有配件
                    ugcprint("SubItemListNum SubIndex <= SubItemListNum")
                    local SubItem = WeaponConfig.SubItemList[SubIndex]
                    SubItem.ItemID = NewID
                    if NewCount and NewCount > 0 then
                        SubItem.Count = NewCount
                    end
                else
                    -- 扩展配件列表
                    ugcprint("SubItemListNum SubIndex > SubItemListNum")
                    if NewCount and NewCount > 0 then
                        for i = SubItemListNum + 1, SubIndex do
                            if i == SubIndex then
                                -- 添加新配件
                                table.insert(WeaponConfig.SubItemList, {ItemID = NewID, Count = NewCount})
                            else
                                -- 添加占位符
                                table.insert(WeaponConfig.SubItemList, {ItemID = -1, Count = -1})
                            end
                        end
                    end
                end
            else
                -- 修改主武器
                WeaponConfig.ItemID = NewID
                if NewCount and NewCount > 0 then
                    WeaponConfig.Count = NewCount
                end
            end
```

### OrganizeWeaponList()

**功能**：清理无效的武器槽位

**处理逻辑**：

1. 获取当前武器列表
2. 清理 LastWeaponListIndex 之后的所有槽位
3. 移除玩家编辑过程中产生的无效数据

**使用场景**：

* 玩家复活时，清理上次编辑留下的无效槽位
* 确保武器列表整洁，避免发放无效装备

```
function UGCPlayerController:OrganizeWeaponList()
    local WeaponList = self.PlayerState.WeaponList
    
    for i = self.LastWeaponListIndex + 1, #WeaponList do
        WeaponList[i] = nil
    end
end
```
