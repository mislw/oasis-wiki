## 语音系统

> 文档路径: 进阶内容 > 和平精英集成功能 > 语音系统

**涉及API:** `OnRep_UGCTeamID`, `UGCPlayerController`, `UGCTeamID`, `UGCTeamID为开发者自己注册的同步参数`, `UGCVoiceManagerSystem`, `UGCVoiceManagerSystem.JoinVoiceRoom`

#  语音系统

### 功能介绍

和平语音系统为和平和绿洲玩法内置功能，主要用于局内玩家语音对话。
该功能进入游戏后自动开启，频道一般分为队伍语音和全局语音，队伍语音会按照进入游戏时的组队关系分配，如果开发者在游戏内对队伍进行了重新分配，那么语音也需要相应的进行分配。

### 接口库

API文档：[UGCVoiceManagerSystem](https://developer.gp.qq.com/api/#/searchContent/UGCVoiceManagerSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%A4%BE%E4%BA%A4%E7%B3%BB%E7%BB%9F%2FUGCVoiceManagerSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCVoiceManagerSystem)

### 使用说明

接口库中最常用的功能为UGCVoiceManagerSystem.JoinVoiceRoom，在队伍关系重新分配后，需要把同队的人分配到同一个房间，可以把玩家的队伍ID传入客户端，然后自行组合RoomKey，如TeamRoom_<span style="color: #EE0000">TeamID</span>。

如在TeamID的OnRep（属性同步）事件中，为玩家分配对应的队伍房间

### 示例代码

```lua
function UGCPlayerController:OnRep_UGCTeamID()				--UGCTeamID为开发者自己注册的同步参数
    local RoomName = "TeamVoiceRoom"..tostring(self.UGCTeamID)
    UGCVoiceManagerSystem.JoinVoiceRoom(RoomName)
end
```

## 好友系统

> 文档路径: 进阶内容 > 和平精英集成功能 > 好友系统

#  好友系统

## 功能介绍：

好友系统是和平集成的社交系统，开发者可以通过在适当的时机让玩家相互添加好友，增加玩家组队游玩热情从而提高玩法留存率。
最常见的添加好友的渠道是在结算界面增加添加好友的按钮，部分更重社交的游戏还可以在其他位置也添加。

## 相关接口

|接口 |描述 |适用范围 |
| ------ | ------ | ------ |
|FriendSystem.IsMyFriend(UID) |返回玩家是否已是好友 | 客户端 |
|FriendSystem.AddFriend(UID, BP_ENUM_ADD_FRIEND_FROM_BATTLE_RESULT, ContentIndex) |申请添加好友，其中ContentIndex，传入7为对男性用语，传入8为对女性用语 | 客户端 |

## 示例代码

```
--添加好友按钮按下
function Button_AddFriend_OnClicked()
    --发送好友申请
    if PlayerSex == 1 then
        FriendSystem.AddFriend(UID, BP_ENUM_ADD_FRIEND_FROM_BATTLE_RESULT, 7); -- 7是在好友申请配置表配置的男性默认语
    else
        FriendSystem.AddFriend(UID, BP_ENUM_ADD_FRIEND_FROM_BATTLE_RESULT, 8); -- 8是在好友申请配置表配置的女性默认语
    end
    PopUpNoticeUI.ShowFastNoticeQueue(string.format("已向%s发送好友申请消息", self.PlayerResultData.PlayerName));
end
```

PS：具体可以参照躲猫猫的结算UI逻辑，有现成的案例。

#  玩家头像

## 功能介绍

部分功能比如结算界面或者实时积分面板可能需要显示玩家当前的头像，从而更容易的识别玩家，这里提供一个和平集成的头像控件，只需要传入特定参数即可。

![企业微信截图_16980302791243.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/lCfHE%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16980302791243.png)

<br>

## 使用说明

创建一个继承自Common_Avatar_BP的控件蓝图（蓝图名需要和父类有所区别），把该UMG拖放到需要显示头像的位置，调用方法InitView。

```lua
Common_Avatar_BP_Widget:InitView(
int Style,
int UID,
string IconURL, 
int Gender,
int FrameLevel,
int PlayerLevel,
bool IgnoreFrame,
bool IsMySelf)
```

Style一般默认传入1。

UID，IconURL，Gender，FrameLevel，PlayerLevel一般可以在PlayerState中直接获取到。

IgnoreFrame为是否显示外框。

IsMySelf为是否自己的头像，会有一些特殊显示。

```
IconURL = PlayerState.IconURL;
Gender = PlayerState.PlatformGender;
PlayerLevel = PlayerState.PlayerLevel;
FrameLevel = PlayerState.SegmentLevel;
```
<br>

## 示例代码

```
self.Common_Avatar_Widget:InitView(1, PlayerResultData.UID, PlayerResultData.IconURL, PlayerResultData.Gender, PlayerResultData.FrameLevel, PlayerResultData.PlayerLevel, true, false);
```

<span style="color: #EE0000">注意事项：PlayerState的相关性为队友相关，在客户端无法获取到非同队玩家的PlayerState，有需要的话可以在DS统计，然后在GameState建一个数据表属性同步下来。</span>

# 绿洲启元段位和赛季接入指南

## 绿洲启元段位和赛季简介

1. 玩法可以根据实际的玩法需求，定制适合玩法自身的段位方案，让玩家在此玩法中能匹配更合适的队友以及对手，也能帮助玩家在玩法中建立更好的长期追求和目标。
2. 与段位配套的还有赛季，玩法可以根据运营需求，设置赛季，并在赛季更替时，定制段位重置规则。

下文中【绿洲启元玩法段位】和【绿洲启元段位积分】将统一简称【段位】和【积分】，区别于和平精英原本的段位和积分，仅单独作用于绿洲启元中的玩法。

<br>

**2. 改变玩家积分**

- 此接口仅在【战斗服务器】生效。
- Count为需要增加或者减少的段位积分数量。

```lua
---修改段位分
---生效范围：S
---@param PlayerKey int  @玩家PlayerKey
---@param Count int  @段位分
```

```lua

function PlayerController：GetUGCPlayerNewSeasonRankInNewGameSeason(GameSeasonId, PlayerSeasonId, PlayerRank)
    ...
    return NewSeasonRank
end
```

### 段位基础接入流程
1. 根据玩法实际场景，调用`UGCRankSystem.AddRanktProgress()`接口,增加或减少玩家的段位分数。

2. 游戏结束时，确保`UGCGameSystem.SendPlayerSettlement()`被调用，否则玩家在本局的积分变化将不被结算。

3. 客户端调用`UGCGameSystem.ShowUGCRankAndAchievementUI()`接口，显示段位结算界面，若本局段位积分变化为0， 也依然会显示段位结算界面。

4. 根据赛季需要，在PlayerController脚本中，重载`PlayerController：GetUGCPlayerNewSeasonRankInNewGameSeason`接口，根据玩家之前赛季id和段位积分，返回新赛季玩家段位积分，若未重载，则默认直接保留玩家之前段位积分。

<br>

## 注意事项
-  `UGCGameSystem.ShowUGCRankAndAchievementUI`需要保证在`UGCGameSystem.SendPlayerSettlement()`调用后再调用,否则结算界面不会正确显示。
-  未游玩过玩法的新玩家不受赛季重置积分影响。

<br>

**FAQ：我的玩法有自己的结算界面，结算流程应该如何接入段位结算界面?**

1. 确保战斗服务器内已调用`UGCGameSystem.SendPlayerSettlement()`接口。

2. 客户端正常显示【自己实现的结算界面】。

3. 结算界面隐藏【返回大厅】按钮。

4. 结算界面中监听委托`UGCGameSystem.NotifyUGCRankAndAchievementUIDelegate`，委托被触发时执行逻辑显示【返回大厅】按钮。

5. 当徽章，段位结算界面关闭，或被判定为不显示情况下，都会触发委托`UGCGameSystem.NotifyUGCRankAndAchievementUIDelegate`。

6. 调用`UGCGameSystem.ShowUGCRankAndAchievementUI`接口显示段位以及徽章结算界面。

## 绿洲启元徽章简介

开发者可以根据实际的玩法需求，定制适合玩法自身的徽章方案，帮助玩家在玩法中建立更好的长期追求和目标。

<br>

## 徽章设计建议

**1. 获取难度需要分层次**
- 建议正常游玩就能自然获得部分徽章，但也需要带有一定挑战性，或者需要去特定挑战才能获得的徽章。
- 建议正常游玩能获得60%左右的徽章，25%需要去完成但难度不高，15%需要花一些时间和精力去挑战收集。

**2. 把你想让玩家体验到的有趣内容做成徽章**
- 除了玩家正常游戏就能体验到的内容，徽章可以包含一些专门设计的彩蛋或者隐藏内容，帮助玩家更明确的了解到游戏内还包含这些有趣的内容。

**3. 不建议设计的徽章类型**
- 枯燥且乏味的大数量收集。
- 需要过高的操作技巧或者极小概率才能遇到的内容。

**徽章进度是否累计的配置项说明**

- **累计型徽章** ：代表玩家可以在多局有游戏中持续累计徽章进度
例：徽章为收集100枚金币，玩家第一局游戏收集了5枚金币，则第一局游戏结束后，此徽章进度为5/100，玩家第二局游戏中收集了10枚金币，则第二局游戏结束后，此徽章进度为15/100。

- **非累计徽章** ：代表玩家需要在单局游戏中达成徽章条件，才获得徽章，多局之间数量不累计
例：徽章依然为收集100枚金币，玩家第一局游戏收集了85枚金币，没有达成100枚金币的收集条件，则第一局游戏结束后，此徽章进度为0/100，玩家第二局中，收集了105枚金币，则第二局游戏结束后，徽章状态为已获得。

## 战斗内徽章系统接入流程

### 核心接口

1. 此接口仅在【战斗服务器】生效。

2.  `AchievementID`为绿洲启元管理平台中项目里配置的徽章ID。

3.  Count为需要增加的徽章计数数量。

```lua
---累积徽章进度
---生效范围：S
---@param PlayerKey int  @玩家PlayerKey
---@param AchievementID int  @徽章ID
---@param Count int  @徽章计数
function UGCAchievementSystem.AddAchievementProgress(PlayerKey, AchievementID, Count)
end
```

### 徽章基础接入流程

1. 在工程中，根据逻辑自行计数徽章进度计数值。

2. 调用`UGCAchievementSystem.AddAchievementProgress()`接口,将1中得到的计数值，传给每个徽章，增加玩家的徽章计数。

3. 游戏结束时，确保`UGCGameSystem.SendPlayerSettlement()`被调用，否则本局累计的徽章进度将不被结算。

4. 客户端调用`UGCGameSystem.ShowUGCRankAndAchievementUI()`接口，显示徽章结算界面，若本局没有新获得的徽章，则徽章结算界面不会弹出。

<br>

## 注意事项

1. `UGCAchievementSystem.AddAchievementProgress`的调用为覆盖模式，即只会保留该徽章最后一次调用接口传进去的数值，所以需要开发者在逻辑中进行统计计算，确保使用`UGCAchievementSystem.AddAchievementProgress`接口传入的值为最终数值。

2. `UGCGameSystem.SendPlayerSettlement`必须在`UGCAchievementSvstem.AddAchievementProgress`之后调用，否则结算界面无法正确显示结果。

<br>

# 信号圈系统

游戏中的信号圈系统将地图划分为安全的圈内与危险的圈外，处于圈外的玩家会随时间降低信号值，信号值降低至0后，玩家会死亡。开发者可以利用信号圈系统，有效限制玩家活动范围、推动对战节奏。

<br>

## 信号圈流程

信号圈系统按照预设阶段顺序严格执行，每个阶段的配置参数将按CircleList数组索引顺序依次生效。开发者需理解其运行规律，保证信号圈的变化符合预期。

## 信号圈配置

进入【玩法通用设置】，从【Active Game Part Configs】找到 ``GP_CircleManager`` 模块，信号圈的配置项以及说明如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RwGTmimage.png)

|属性名称|说明|
|:-:|-|
|Start On Enter Fight|是否开局启动信号圈|
|Init Blue Circle Point|初始信号圈中心点，单位：厘米|
|Init Blue Circle Radius|初始信号圈半径，单位：厘米|
|Circle List|每阶段的信号圈配置信息列表|

信号圈配置信息列表中，每个数组元素对应信号圈的一个阶段，信号圈的变化按配置列表数组顺序依次执行，每一项的数组元素的配置说明如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/nhqZsimage.png)

|属性名称|说明|
|:-:|-|
|Pain|信号圈外玩家每秒减少的信号值，最低为0，当次的缩圈预警出现时，伤害值修改并生效|
|DelayTime|延时阶段时长，单位：秒，最低为0|
|BlueCirclePreWarning|缩圈预警阶段设置（单位：秒）<br>• 数组元素需从小到大排列<br>• 取数组中大于30的最大值作为预警阶段时长，若无符合条件的元素，预警时长默认为30秒<br>• 其余元素表示经过对应秒数时触发预警提示的时间点|
|LastTime|缩圈阶段时长，即从开始缩圈到缩圈结束的时间，单位：秒，最低为0|
|WhiteRadius|白圈半径，即结束缩圈时的信号圈半径，单位：厘米，最低为0|
|bUseCustomWhitePoint|是否自定义白圈中心，即是否自定义结束缩圈时的信号圈中心点，勾选时使用【WhitePoint】指定坐标，未勾选时随机生成中心点|
|WhitePoint|自定义结束缩圈时的中心点坐标，勾选【bUseCustomWhitePoint】时生效|


信号圈系统仅支持精确控制信号圈的缩小，并且必须是缩小后的信号圈完全包含在缩小前的信号圈范围内。若检测到不合理的缩小信号圈配置或进行信号圈的扩大，系统只能精确变化信号圈大小，中心位置随机变化。

``` lua
--MainUI.lua部分代码

--UI控件的初始化函数
function MainUI:Construct()
	--给按键绑定点击事件回调
	self.ButtonStartCircle.OnClicked:Add(self.ButtonStartCircle_OnClicked, self);
	self.ButtonStopCircle.OnClicked:Add(self.ButtonStopCircle_OnClicked, self);
end

function MainUI:ButtonStartCircle_OnClicked()
	--获取客户端当前的 PlayerController
  	local PlayerController = UGCGameSystem.GetLocalPlayerController();
	--调用PlayerController的ServerRPC函数
  	UnrealNetwork.CallUnrealRPC(PlayerController, PlayerController, "StartCircle");
	return nil;
end

function MainUI:ButtonStopCircle_OnClicked()
	--获取客户端当前的 PlayerController
  	local PlayerController = UGCGameSystem.GetLocalPlayerController();
	--调用PlayerController的ServerRPC函数
  	UnrealNetwork.CallUnrealRPC(PlayerController, PlayerController, "StopCircle");
	return nil;
end


--UGCPlayerController.lua部分代码

--接收RPC调用
function UGCPlayerController:GetAvailableServerRPCs()
    return
    "StartCircle",
    "StopCircle"
end

function UGCPlayerController:StartCircle()
    --开启信号圈
    UGCCircleManagerSystem.StartCircle()
end

function UGCPlayerController:StopCircle()
    --关闭信号圈
    UGCCircleManagerSystem.StopCircle()
end
```

# 空投系统

空投系统是游戏内重要的物资投放机制，可在指定位置生成包含各类物资的空投箱，为玩家提供额外的战略资源和战术选择，开发者可灵活配置空投箱内的物资内容，利用空投系统，丰富游戏过程中的资源争夺与策略博弈。

<br>

## 电磁干扰区系统

> 文档路径: 进阶内容 > 和平精英集成功能 > 电磁干扰区系统

**涉及API:** `BP_EMPZoneControlComp`, `BP_UGCEMPZoneActor`, `DestroyElectromagneticArea`, `GP_EMPZone`, `GenerateElectromagneticArea`, `UGCGameMode`

# 电磁干扰区系统

电磁干扰区是游戏内的战术环境机制，可在指定区域生成持续性的电磁干扰效果，该效果可导致载具熄火、瞄具准星消失，并使玩家屏幕出现视觉花屏干扰，开发者可以利用电磁干扰区系统，剥夺范围内玩家的机动性与作战效能，丰富游戏的战术选择和策略深度。

<br>

## 电磁干扰区流程

电磁干扰区系统按照以下流程执行相应的阶段行为：

![信号圈系统-电磁干扰区流程.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/R3yaT%E4%BF%A1%E5%8F%B7%E5%9C%88%E7%B3%BB%E7%BB%9F-%E7%94%B5%E7%A3%81%E5%B9%B2%E6%89%B0%E5%8C%BA%E6%B5%81%E7%A8%8B.drawio.png)

1. 生成指令下达后，电磁干扰区不会立即显现，而是进入一段准备时间，区域内暂无电磁干扰效果
2. 准备时间结束后，电磁干扰区出现并生效，指定区域出现电磁干扰效果，同时小地图会显示电磁干扰区的位置与范围
3. 当设定的持续时长结束时，电磁干扰区效果自动消失，受影响区域内的一切功能将恢复正常

## 轰炸区系统

> 文档路径: 进阶内容 > 和平精英集成功能 > 轰炸区系统

**涉及API:** `GP_AirAttack`, `GenerateBombingArea`, `StopBombingArea`

# 轰炸区系统

轰炸区是游戏内的战术环境机制，可对指定区域持续投放炸弹并造成打击，开发者可灵活配置轰炸区的范围、时长、伤害参数等属性，利用轰炸区系统实现对特定区域的控制或高难度挑战，从而丰富游戏的战术维度和策略深度。

<br>

## 轰炸区流程

轰炸区系统按照以下流程执行相应的阶段行为：

![信号圈系统-轰炸区流程.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/QgVMP%E4%BF%A1%E5%8F%B7%E5%9C%88%E7%B3%BB%E7%BB%9F-%E8%BD%B0%E7%82%B8%E5%8C%BA%E6%B5%81%E7%A8%8B.drawio.png)

1. 生成指令下达后，轰炸区进入预警阶段，区域内暂未发生实际轰炸，小地图会以特定颜色显示轰炸区的位置与范围
2. 预警阶段结束后，轰炸区进入轰炸阶段，轰炸区内落下炸弹，小地图上的区域提示颜色将切换为另一种颜色
3. 设定的轰炸阶段结束时，所有轰炸行为立即停止，轰炸区效果完全消失，小地图上对应的区域提示也随之清除

<br>

## 轰炸区配置

进入【玩法通用设置】，从【Active Game Part Configs】找到 ``GP_AirAttack`` 模块，点击【Air Attack Configs】右侧的加号，添加新的轰炸区的配置项。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/teT0Vimage.png)

轰炸区的配置项以及说明如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/B7DBEimage.png)

|属性名称|说明|
|:-:|-|
|EscapeTime|轰炸区预警阶段时长，此阶段结束后进入轰炸阶段，单位：秒，范围：[10,180]|
|AttackLastingTime|轰炸区轰炸阶段时长，单位：秒，范围：[10,180]|
|AttackAreaRadius|轰炸区作用半径，单位：厘米|
|TotalBombCount|轰炸阶段炸弹投放总数，范围：[1,50]|
|BombMaxDamage|炸弹爆炸的中心范围伤害值，范围：[10,500]|
|BombMinDamage|炸弹爆炸边缘的最小伤害值，范围：[10,500]|
|BombRadius|单颗炸弹的爆炸影响半径，单位：厘米|
|CenterDamageRange|炸弹中心伤害范围，超出此范围伤害将衰减至最小伤害，单位：厘米|
|BombFallSpeed|炸弹下落速度（从1000米高空开始坠落），单位：厘米/秒，范围：[100,1000]|

下方【Air Attack】属性组中可以配置轰炸区在小地图的显示效果，相关配置项以及说明如下：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/cC5BUimage.png)

|属性名称|说明|
|:-:|-|
|Warning Color|预警阶段在小地图上显示的提示颜色|
|Bombing Color|轰炸阶段在小地图上显示的提示颜色|
