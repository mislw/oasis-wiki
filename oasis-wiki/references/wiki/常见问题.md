## 编辑器安装

> 文档路径: 常见问题 > 编辑器安装

## Q1：编辑器登录失败，弹窗内容显示乱码，该如何解决

一般为网络权限问题，检查是否被电脑管家，360等防护软件拦截。

<br>

## Q2：编辑器启动失败或者出现闪退现象，一般是什么问题引起的

检查编辑器的安装路径，为确保编辑器的功能使用正常，WeGame及绿洲启元编辑器的安装目录路径，请勿包含空格、括号等特殊符号（仅支持英文及下划线）。

## 玩法流程

> 文档路径: 常见问题 > 玩法流程

## Q1：游戏运行时出现较大的网络延迟、波动甚至断开服务器，该如何排查

该类问题主要由于网络流量负载过大造成，可从两方面排查：
- RPC：检查可靠RPC调用的数量和频率，大量的可靠RPC调用会导致网络带宽和延迟的压力，特别是在需要保证每个调用都成功的情况下。可尽量减少不必要的可靠RPC，或者替换使用不可靠RPC
- 属性同步：属性同步目前只支持Lua的数据类型，即bool、number（int，float）、string、table以及nil几种类型，其他的如actor、component等UE对象类型，以及FVector、FHitResult等UE结构体，都不可以直接使用属性同步，在同步之前需先将数据转为table形式

<br>

## Q2：管理平台观察到玩法在灰度/现网中出现很多DS超时，应该如何处理

首选需要确保对局流程在DS有效时长内能正确结束，且结束后需要主动关闭DS，防止DS进程空跑，具体步骤可参考 [游戏结束时的逻辑处理](https://developer.gp.qq.com/wikieditor/#/catalog/272)。

## GamePlay

> 文档路径: 常见问题 > GamePlay

## Q1：如何开启队伍语音

和平语音系统是和平为绿洲玩法内置的功能，需要使用 [UGCVoiceManagerSystem](https://developer.gp.qq.com/api/#/searchContent/UGCVoiceManagerSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%A4%BE%E4%BA%A4%E7%B3%BB%E7%BB%9F%2FUGCVoiceManagerSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCVoiceManagerSystem) 函数库动态启用，具体查看 [语音系统](https://developer.gp.qq.com/wikieditor/#/catalog/212)。

<br>

## Q2：如何修改玩家角色的形象

- 更换模型：技能编辑器提供了 [变身Buff模板](https://developer.gp.qq.com/wikieditor/#/catalog/20253)，可通过释放Buff来实现变身效果，但是仅支持变身为主角骨骼的模型；也支持在脚本中调用 [ChangeAvatarMesh](https://developer.gp.qq.com/api/#/searchContent/UGCPlayerPawnSystem?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCPlayerPawnSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCPlayerPawnSystem&autoJump=ChangeAvatarMesh) 动态切换模型，同样只支持主角骨骼
- 角色隐身：技能编辑器提供了 [隐身Buff模板](https://developer.gp.qq.com/wikieditor/#/catalog/20253?autoJump=%E9%9A%90%E8%BA%AB)，可通过释放Buff来实现隐身效果；也支持在脚本中调用 [HideBoneByBoneName](https://developer.gp.qq.com/api/#/searchContent/UGCPlayerPawnSystem?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUGCPlayerPawnSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCPlayerPawnSystem&autoJump=HideBoneByBoneName) 方法隐藏指定的骨骼部位
- 角色缩放：目前不建议修改玩家主角骨骼的缩放（例如巨大化效果），容易造成穿模及变形，且影响主角的动画表现

<br>

## Q3：是否可以为玩家角色添加类似翅膀这样的额外模型

可以添加模型，但是需要通过物品编辑器创建头、包、甲三类物品，更换物品模型来间接实现，因为和平avatar皮肤多样性和复杂性，无法直接基于骨骼挂点的方式添加模型，容易出现穿模的问题。

<br>

## Q4：编辑器不支持渲染导出关卡序列动画，那么如何播放关卡序列以及Actor序列

- 关卡序列：播放关卡序列需继承新建LevelSequenceActor蓝图作为播放器，将关卡序列导入而后播放，具体操作请查看 [Sequence动画](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/cinematics-and-sequencer?application_version=4.27)。
- Actor序列：Actor序列使用的是ActorSequenceComponent组件，可使用StartPlay以及StopPlay操作播放，如self.actorSequence:StartPlay(0, true)

``` lua
---@field StartPlay:fun(StartTime:float,bOpenTick:bool)
---@field StopPlay:fun()
```

<br>

## Q5：怪物在寻路时为什么会直接穿过墙壁移动，而不是绕过障碍物

怪物的寻路避障依赖于寻路网格，需要先构建寻路网格，才可以进行避障，具体操作可查看 [使用导航网格(NavMesh)](https://developer.gp.qq.com/wikieditor/#/catalog/177)。

<br>

## Q6：被动技能通常在事件触发后执行多种行为，因此需要配置多项行为Task，如何修改Task的顺序以快速调整技能

每个行为配置项前有锚点控件，选中并按住对应的行为即可拖拽变更顺序。

## Q7：如何实现角色一边移动一边释放技能的效果

默认配置的状态互斥表对移动状态设定了打断和禁用规则，需要 [修改互斥规则](https://developer.gp.qq.com/wikieditor/#/catalog/20106?autoJump=%E9%85%8D%E7%BD%AE%E4%BA%92%E6%96%A5%E8%A7%84%E5%88%99) 来实现。

<br>

## Q8：技能中带有生成Actor的技能Task，但是为什么技能实际释放效果没有生成对应的Actor

由于技能实例在DS端运行，客户端模拟，需要确保Actor开启了Replicates复制属性。

<br>

## Q9：技能机制是先选取指定目标而后造成伤害，为什么明明选中了目标却没有造成伤害

技能按照时间轴上编排的Task先后顺序执行，需要确保选择目标Task在造成伤害Task之前执行完成。

<br>

## Q10：对两个玩家设置了同一个阵营，为什么仍会互相造成伤害

阵营和队伍是两个独立的系统，同阵营但不同队伍也是存在伤害的。

<br>

## Q11：玩家登录游戏后UGCPlayerPawn第一次创建生成的时候，会在客户端连续初始化很多次，是否属于正常现象

因为和平精英使用了对象池的优化技术，所以客户端会在第一次创建UGCPlayerPawn的时候生成多个实例对象，因此会触发多次ReceiveBeginPlay，开发者在处理角色相关业务逻辑时，需要注意这个特性。

<br>

## Q12：通过AddItemV2将平底锅添加到背包，并且使用EquipItemV2让玩家装备该平底锅，但是实际表现是平底锅只加在背包里没拿到手上，日志中也没有执行报错信息，这是什么原因

新背包对和平经典物品只做了基础兼容处理，不保证可用性完全一致，因此更适用于新物编创建的物品，建议开发者将新物编与新背包配套使用。

<br>

## Q13：使用AddItem给玩家添加自定义的武器时，会出现给予失败的情况，可能是什么原因造成的

新旧物编配备的函数库是独立的，对于用新物编创建的物品必须使用UGCBackpackSystemV2的函数 [AddItemV2](https://developer.gp.qq.com/api/#/searchContent/UGCBackpackSystemV2?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%89%A9%E5%93%81%E4%B8%8E%E8%83%8C%E5%8C%85%2FUGCBackpackSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCBackpackSystemV2&autoJump=AddItemV2)，建议开发者遇到此类问题时，先检查创建物品所使用的物编版本和函数库是否一致。

<br>

## Q14：燃烧瓶可以正常对玩家造成伤害包括燃烧的粒子效果，但是灼烧怪物并不会触发伤害回调，燃烧瓶的伤害对象是不是没有支持到怪物

新物品编辑器创建的燃烧瓶不支持对旧怪物基类的伤害，建议开发者使用实体编辑器创建怪物。

<br>

## Q15：如何隐藏背包的入口按钮

- 新背包需在【玩法通用设置】设置是否显示背包按钮，也可以通过调用 [SetBackpackButtonVisible](https://developer.gp.qq.com/api/#/searchContent/UGCBackpackSystemV2?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E7%89%A9%E5%93%81%E4%B8%8E%E8%83%8C%E5%8C%85%2FUGCBackpackSystemV2.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCBackpackSystemV2&autoJump=SetBackpackButtonVisible) 函数控制背包按钮的动态显隐
- 经典背包需采用widgetlayout方案，变更MainUI_BackPack控件的可视性为“隐藏”或“已折叠”来实现

<br>

## Q16：背包内添加的物品数据能否跨对局持久化存储

新物品编辑器创建的物品可以通过启用【是否持久化】属性来实现；旧物资编辑器创建的物品不支持持久化存储功能，可通过玩家存档数据来间接存储。

<br>

## Q17：怪物的攻击行为包括动画是在哪里配置的,如何修改怪物攻击表现

怪物的攻击本质上是释放攻击技能，因此根据行为树中施放技能Task执行的技能槽位，修改怪物蓝图PersistBaseComponent组件中对应槽位绑定的技能蓝图来实现。

<br>

## Q18：载具能否更换材质或贴图

所有载具目前暂时不支持换材质/贴图。

<br>

## Q19：为了方便测试，希望给玩法添加一些GM功能，应该如何实现

编辑器为玩法运行时内置了 [GM面板](https://developer.gp.qq.com/wikieditor/#/catalog/20109)，可以在该框架下自定义添加任意GM功能，此面板仅对内网生效，不影响正式环境。

<br>

## Q20：为什么技能释放时被无故打断，为什么受到攻击时无法释放技能

由于技能模板使用官方配置的 [默认状态互斥表](https://developer.gp.qq.com/wikieditor/#/catalog/20106?autoJump=%E4%BA%92%E6%96%A5%E8%A7%84%E5%88%99%E8%A1%A8)，如果命中互斥规则时即会出现此类现象，开发者可优先基于互斥表排查问题，必要时可按需修改互斥规则。

## 上传&调试

> 文档路径: 常见问题 > 上传&调试

## Q1：工程完成后，如何在手机上进行测试

1. 目前真机测试只支持安卓手机，需先安装手机客户端，将pak包放入安卓手机安装即可，pak包路径为：``…\WeGameApps\rail_apps\和平精英绿洲启元编辑器(2001776)\安卓``
2. 安装完成后，开发者需将工程上传，分为测试上传与发布上传，其中测试上传的版本仅用于内部的玩法流程测试；发布上传的版本可用于部署到正式服，可以模拟玩法在和平精英绿洲启元大厅的上架状态，适用于更接近实际发布环境的测试。详细内容可查看 [上传工程与真机测试](https://developer.gp.qq.com/wikieditor/#/catalog/372)

<br>

## Q2：真机测试时DS日志显示在加载资源对象时报错，但客户端日志正常，这种情况是什么原因导致的

工程cook时不会将贴图、材质等表现类资源打进DS包，因此在DS端执行这类资源加载会报错，需要确保表现类逻辑只在客户端执行。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/iePz9image.png)

<br>

## Q3：工程上传后为什么在移动端上找不到该玩法

工程上传后不显示玩法可能有以下原因，需要逐一排查：
1. 上传包含工程包烘焙和服务器推送等多个环节，上传完成只代表烘焙结束，可能存在延迟显示的情况，通常时间间隔在10分钟内
2. 编辑器版本与和平精英外研线客户端版本号的前三位要保持一致（如：1.34.X），当编辑器新版本发布且wegame自动更新后上传的工程，如果客户端未同步更新则会因版本不一致而无法显示玩法
3. 上传后的玩法工程包存在定时清理策略，超过一周的玩法版本将被自动清理

<br>

## Q4：真机测试发现贴图/材质变白模，甚至有些场景中的物件资源不加载，但是PIE调试正常，这种情况如何解决

如果出现异常的资源为资源目录下的公共资源，则可能被和平精英标记为旧资源而剔除主包导致，可以尝试利用 [Lua资源引用](https://developer.gp.qq.com/wikieditor/#/catalog/314?autoJump=%E9%80%9A%E8%BF%87Lua%E8%B5%84%E6%BA%90%E5%BC%95%E7%94%A8%E6%8C%87%E5%AE%9A%E8%B5%84%E6%BA%90) 的方式解决；如果为工程内导入的资源，则可能因cook问题导致，可以尝试重新上传解决。

<br>

## Q5：通过画刷工具绘制的地表贴图，PIE调试时正常显示，但真机测试时被还原成默认材质是什么原因

刷地形需要变更地表材质引用的贴图资源，因此需要替换landscape引用的地表材质实例，详细内容可查看 [地表材质使用介绍](https://developer.gp.qq.com/wikieditor/#/catalog/300?autoJump=%E5%9C%B0%E8%A1%A8%E6%9D%90%E8%B4%A8%E4%BD%BF%E7%94%A8%E4%BB%8B%E7%BB%8D)。

<br>

## Q6：为什么同样的场景，编辑器视口包括PIE调试时显示的光照效果和真机测试效果不同？

由于PIE调试的渲染管线与真机的渲染管线不同，所以存在差异，目前需要调整光照来达到预期效果。

<br>

## Q7：场景中的点光源在PIE调试时正常，但是打包到真机上就丢失了，是不支持点光源功能吗

目前确实不支持点光源/聚光源功能，需要等待后续新版本支持，可使用【天气控件】处理游戏光照，详细内容可查看 [天气系统](https://developer.gp.qq.com/wikieditor/#/catalog/225)。

<br>

## Q8：编辑器是否有配备测试用的IOS安装包，如何自测玩法在IOS上的表现效果

目前仅提供了安卓测试包，玩法提审后内部测试会覆盖IOS并提供测试结果反馈，开发者自测阶段无需关注IOS端。

## 商业化

> 文档路径: 常见问题 > 商业化

## Q1：抽奖模板奖池表的LotteryRule字段填写长文本时，如何换行

从下载的模板目录 ``ExtendResource/Lottery/OfficialPackage/Asset/Lottery/Blueprint/Struct/Table`` 中找到表格对应的结构体LotteryData，将结构体中LotteryRule字段的“多行文本”属性勾选上，然后在编辑字段值时通过Shift+Enter键实现换行。其他模板涉及表格编辑长文本字段的场景以同样的方式处理。

<br>

## Q2：玩法开通了内购权限且接入了商业化功能，工程上传后点击购买提示“商品未上架，无法购买”是什么原因

确保玩法已开通内购权限后，需确认玩法的测试环境：
- 如果是PIE调试，商品默认都能成功购买，出现提示则检查商品的上下架时间是否正确
- 如果是真机测试，确认使用的是“测试上传”还是“发布上传”，测试上传默认购买失败；发布上传出现提示，则检查商品的上下架时间是否正确

## UI

> 文档路径: 常见问题 > UI

## Q1：为什么在移动、滑动或操作其他控件时，无法正常点击UI按钮

这通常是因为按钮或其他控件的 IsFocusable 属性被启用，导致它们在接收输入焦点时与其他操作冲突。关闭该属性可以避免输入被拦截，从而恢复按钮响应。

<br>

## Q2：如何隐藏【退出观战】控件

适当的时机下在PlayerController的客户端执行 ``self:CastUIMsg("UIMsg_HideQuitWatch", "")``。

<br>

## Q3：如何为玩法添加和平精英主模式的击杀播报UI

在GameMode蓝图中配置，找到GMDataManager - Global Settings - UI模块预置，将KillInfoModule填入。如果有多个模块需要填入，可用|符分割。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/2OyzWimage.png)

<br>

## Q4：如何将图片资源加载并显示在Image控件中

相关代码如下：

``` lua
-- 加载纹理资源
local Texture = UGCObjectUtility.LoadObject(imagePath)
-- 将纹理资源设置到Image控件的Brush属性中
self.Image:SetBrushFromTexture(Texture, true)
```

<br>

## Q5：为物件对象蓝图添加了Widget组件，为什么运行时更新绑定的UI控件变量值，但是控件效果却无法正常响应变化

使用 ``Widget:RequestRedraw()`` 来强制Widget刷新并重新绘制UI控件，这个方法会触发Widget重新渲染，确保UI控件根据最新数据显示。

<br>

## Q6：如何解决自定义的UI控件被和平原生控件遮挡的问题

由于和平原生控件分散在不同的控件蓝图中，ZOrder属性无法解决跨控件蓝图的层级问题，推荐使用控件锚点的方案，使用锚点可以将自定义的控件添加到指定层级，结合控件布局实现更精准的挂载效果，解决被和平控件遮挡的问题，具体操作请查看 [和平控件锚点](https://developer.gp.qq.com/wikieditor/#/catalog/20097)。

<br>

## Q7：如何隐藏主界面上的和平精英原生控件

## Q3：如何获取玩家的Pawn，PlayerController以及PlayerState

获取玩家的Pawn：
1. 玩家的 Pawn 通常是通过 PlayerController 访问的，因为每个 PlayerController 都控制一个 Pawn，可以使用 PlayerController 获取玩家的Pawn，方法：``PlayerController:GetPlayerCharacterSafety()``，客户端与服务端皆可
2. 通过 PlayerKey 获取玩家的Pawn：``UGCGameSystem.GetPlayerPawnByPlayerKey(PlayerKey)``，在客户端服务端皆可
3. 在客户端获取玩家的Pawn：``UGCGameSystem.GetLocalPlayerPawn()``、``GameplayStatics.GetPlayerCharacter(gameState, 0)``，GetPlayerCharacter方法第一个参数为WorldContextObject，它提供了在【哪个世界】 中查找或执行操作的信息，所以此处需传入一个UObject对象，推荐传入gamestate


获取玩家的PlayerController：
1. 通过玩家的Pawn获取PlayerController：``Pawn:GetPlayerControllerSafety()``、``UGCGameSystem.GetPlayerControllerByPlayerPawn(PlayerPawn)``，客户端与服务端皆可
2. 通过 PlayerKey 获取玩家的PlayerController：``UGCGameSystem.GetPlayerControllerByPlayerKey(PlayerKey)``，支持服务端与客户端
3. 通过 UID 获取玩家的 PlayerController：``UGCGameSystem.GetPlayerControllerByUID(UID)``，仅支持服务端
4. 客户端获取玩家的PlayerController：``UGCGameSystem.GetLocalPlayerController()``、``GameplayStatics.GetPlayerController(gameState, 0)``


获取玩家的PlayerState：
1. 通过PlayerController获取玩家的PlayerState，直接访问属性PlayerState，方法：``PlayerController.PlayerState``，客户端服务端皆可
2. 通过 PlayerKey 获取 PlayerState：``UGCGameSystem.GetPlayerStateByPlayerKey(PlayerKey)``， 客户端服务端皆可
3. 通过 UID 获取玩家的 PlayerState：``UGCGameSystem.GetPlayerStateByUID(UID)``， 仅支持服务端
4. 客户端获取玩家的 PlayerState：``UGCGameSystem.GetLocalPlayerState()``

## Q7：API手册中的RankingListManager与排行榜模板中的RankingListManager有什么区别，如何调用排行榜中的UpdateScore接口

模板中的RankingListManager是静态类，用于将模板里声明的函数统一封装起来；而API手册中的 [RankingListManager](https://developer.gp.qq.com/api/#/searchContent/RankingListManager?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%95%86%E4%B8%9A%E5%8C%96%E4%B8%8E%E5%8A%9F%E8%83%BD%E6%A8%A1%E6%9D%BF%2FRankingListManager.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=RankingListManager) 实际上是排行榜的GamePart模块化派生类，概念上为排行榜功能模块的全局管理器，由于内部实现的原因，两者命名相同因此容易被混淆。

以UpdateScore函数为例，调用示例如下：

``` lua
local GamePartActorInst = UGCGamePartSystem.GetGamePartGlobalActor("RankingListManager");
GamePartActorInst:UpdateScore(PlayerController, 10001, 1, 100, true)
```

除了排行榜模板外，其他模板也存在类似设计，开发者可以简单理解：
- 模板中封装的函数可以通过静态函数的方式直接调用
- GamePart类函数需要获取到模板对应的管理器实例，以对象函数的形式调用

<br>

## 上传失败排查指南

> 文档路径: 常见问题 > 上传失败排查指南

# 上传失败排查指南

> 上传失败后，开发者通常只能看到失败状态，没有详细日志。本文档按最常见到最少见的顺序列出可能原因，逐一排查即可。

---

## 按频率排查（从高到低）

### 1. ReferenceAssetList 配置了不存在的资源

**出现频率最高**

`ReferenceAssetList` 中引用的资源已被删除、重命名或从未导入，导致 Cook 找不到该资源。

**排查方法**：

- 打开 UGC 编辑器，找到 `ReferenceAssetList` 配置
- 逐条检查其中引用的所有资源，确认它们在你的工程中确实存在
- 特别注意：刚删除过资源、重命名过资源、或从其他项目迁移过内容

**修复**：删除不存在的资源引用，或重新导入对应资源后保存。

---

### 2. 引用了其他工程的资源

**出现频率较高**

UGC 项目之间资源隔离，不允许引用其他工程的资源。如果你的 `ReferenceAssetList` 或其他配置中引用了别的项目的路径，就会失败。

**排查方法**：

- 打开 `ReferenceAssetList`，检查所有引用路径是否都以你自己的工程路径开头

**修复**：删除所有非本工程的资源引用，或联系官方确认正确的资源共享方式。

---

### 3. 内存溢出（OOM）— 项目资源过大

**出现频率中等，但影响最严重**

项目资源总量太大，超出了 Cook 节点的内存限制，可能直接崩溃。

**排查方法**：

- 检查你的项目资源总量是否明显偏大（高分辨率贴图多、大模型多、音视频资源多）
- 检查蓝图中是否存在大量**硬引用**（Hard Reference）
- 如果此前 Cook 成功过，回顾最近是否新增了大量资源

**修复**（按优先级）：

1. 清理未使用的资源和重复资源
2. 压缩大体积资源：贴图降低分辨率、模型使用 LOD、音视频降低码率
3. 蓝图中的硬引用改为**软引用**（Soft Reference），改用**异步加载**

---

### 4. DataTable 行结构体缺失

**出现频率较低**

DataTable（数据表）的 `RowStruct`（行结构体）引用丢失，通常是结构体被删除或重命名导致。

**排查方法**：

- 检查项目中所有 DataTable，确认每个 DataTable 的 RowStruct 都指向一个有效的结构体
- 特别注意最近是否删除或重命名过结构体

**修复**：重新创建对应结构体，或将 DataTable 的 RowStruct 修改为正确的结构体。

---

### 5. Shader 编译崩溃

**出现频率较低，常伴随问题 2（跨工程引用）**

Shader 编译进程崩溃，通常是自定义材质节点异常或同时存在跨工程引用导致。

**排查方法**：

- 先检查是否存在问题 2（跨工程引用），优先修复
- 检查项目是否有自定义 Shader 或特别复杂的材质节点
- 材质实例数量是否过多

**修复**：简化复杂材质节点，减少材质实例数量，清理跨工程引用。

---

## 排查建议

1. **先排查 1 和 2**（ReferenceAssetList 相关），这两个占了 65% 以上的失败
2. **再排查 3**（资源量），尤其项目资源较多的团队
3. **最后排查 4 和 5**（DataTable、Shader），出现较少但也要确认
4. 注意：很多失败是**多个原因叠加**，比如内存溢出 + 跨工程引用，需要全部修完再提交

> 以上步骤都排查完仍失败？联系官方技术支持。

## PIE调试失败原因与解决方案

# PIE调试失败原因与解决方案

当 PIE 调试失败时，请在 Output Log 中搜索报错日志关键字，匹配下方对应条目，按建议方案排查修复。若多次尝试仍无法解决，请联系官方技术支持。

---

## 一、入口阶段

### `PIE debug failed: remaining debug count is zero, cannot start debug`

每日调试次数已用完，请等待次日次数自动恢复。

### `PIE debug failed: debug precondition check failed`

调试前置条件检查未通过。请确认已登录账号且工程已成功打开。

### `PIE debug failed: invalid type of asset detected, please check output log`

工程中存在不被支持的资产类型。请打开 Output Log 查看具体资产路径，修复或删除对应资产。

### `PIE debug failed: failed to create debug game thread, UploadPath=...`

创建调试线程失败。请检查磁盘剩余空间是否充足，并确认上传目录具有读写权限。

### `PIE debug failed: SubModeID=... does not exist in project settings`

日志中的 SubModeID 在项目设置中不存在。请检查子模式（SubMode）配置，确保工程配置与代码中激活的 SubModeID 一致。

### `PIE debug failed: copy project to upload directory failed, UploadPath=...`

拷贝项目到上传目录失败。请检查磁盘剩余空间和上传目录读写权限是否正常。

### `PIE debug failed: upload file name validation failed`

上传文件名校验未通过。工程中可能存在包含特殊字符（如中文、空格）或路径过长的文件，请将其改为仅含英文、数字和下划线的短文件名。

### `PIE debug failed: lua file validation failed, debug interrupted`

Lua 文件校验失败，调试已中断。请查看 Output Log 中的 Lua 错误详情，修复对应脚本中的语法或逻辑错误。

### `PIE debug failed: survive folder invalid, SurviveDir=..., default choice is trunk/Survive`

Survive 目录无效。请在 DS 设置中重新选择正确的 Survive 目录路径。

### `PIE debug failed: WinServer folder invalid, ServerWorkingDir=...`

WinServer 目录无效。请在 DS 设置中重新选择正确的 WinServer 目录路径。

### `PIE debug failed: WinServer exe invalid, ServerExePath=...`

## 三、压缩与上传

### `PIE debug failed: zip failed, ...missing umap or ugcproj file, or folder is read-only`

压缩失败。请检查工程中是否缺少 `.umap` 地图文件或 `.ugcproj` 工程文件，并确认文件夹未被设为只读属性。

### `Compress failed: 7z process creation failed, A7zPath=...`

7z 压缩进程创建失败。请检查日志中 A7zPath 路径下 7z.exe 是否存在，若不存在请修复或重新安装编辑器。

### `Compress failed: 7z returned error code, ReturnCode=..., ZipName=...`

7z 压缩返回错误码。请检查磁盘剩余空间是否充足，并关闭可能占用目标文件的其他程序后重试。

### `Compress failed: zip integrity check failed, ZipName=...`

压缩包完整性校验失败。请检查磁盘空间并清理临时文件后重试。

### `Compress failed: missing required ugcproj file`

缺少必需的 `.ugcproj` 工程文件。请确认工程根目录下存在该文件，如缺失请从备份恢复或重建工程。

## 常见问题速查

| 问题类型 | 快速建议 |
|----------|----------|
| **权限问题** | 检查是否完成人脸验证，确认账号有调试/开发/上传权限，必要时联系官方 |
| **网络问题** | 检查网络连接和防火墙，切换更稳定网络后重试 |
| **磁盘/文件问题** | 检查磁盘空间、文件读写权限、文件是否存在或被占用 |
| **工程配置问题** | 检查 SubMode 配置、DS 目录和可执行文件路径 |
| **资产/文件错误** | 检查无效资产类型、非法文件类型、不合法 API 调用 |
| **Lua 脚本错误** | 查看 Output Log 修复 Lua 语法或逻辑错误 |
| **版本问题** | 更新编辑器至最新或推荐版本 |
| **频率/次数限制** | 等待时间间隔或次日恢复 |
| **文件名问题** | 修正文件名中的特殊字符或超长路径 |
| **DS 端问题** | 查看 DS 日志定位崩溃原因 |
| **封禁/限制** | 联系官方申诉 |

---

> 排查 PIE 失败时，建议优先在 Output Log 中搜索 `PIE debug failed` 定位具体报错，再对照本文档查找对应解决方案。如按文档操作后问题仍未解决，请联系官方技术支持。
