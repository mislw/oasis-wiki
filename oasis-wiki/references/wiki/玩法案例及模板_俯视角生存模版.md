### 怪物组表

怪物组表用于对怪物进行自定义编组，并在刷怪管理器中进行配置使用。每个怪物组可以包含一种或多种怪物类型，具体如何划分取决于实际设计需求。

当需要在同一波次中刷新多种怪物时，可以有两种实现思路。一种是将每个怪物组只配置为单一怪物类型，然后在刷怪管理器中通过多个刷新点分别选择不同的怪物组，从而组合出一波包含多种怪物的效果。另一种是直接在一个怪物组中配置该波次需要出现的全部怪物类型，在管理器中只需选择这个怪物组即可完成配置。

样例工程采用的是第一种方式，因此示意图中每个怪物组只对应一种怪物。但实际上，怪物组支持包含多种怪物，具体使用哪种方式可以根据项目需求灵活选择。
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/2D4PFimage.png)

怪物组ID：可自由设定ID，与其他地方对应即可
怪物列表：该组内出现的怪物配置
怪物阵营：同上
怪物类型：可展开选择，对应工程内可用的怪物

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/yJIRrimage.png)

怪物权重：若该组内有多种怪物，且该波次会同时刷新多只怪物，则通过权重控制怪物刷新比例
出生属性：可设定怪物刷新出来时的部分属性。会被怪物表的属性配置覆盖

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/R5Xiuimage.png)

### 刷新器

刷新器可以拖动放置在地图上任意位置，表示对应刷新器下控制的怪物会从该位置刷新出现。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/7VxhRimage.png)

场景大纲视图中，前缀为“BP_Spawner”的对象为怪物的刷新点，其中每个刷怪点都是需要使用”管理器控制“模式，并且使用怪物组表配置。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/19ZChimage.png)

因为刷怪管理器中会对每个刷怪器中的配置进行重写，故每个刷怪器中都没有配置怪物组ID。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/l3Ck3image.png)

怪物阵营：对应怪物组表中的阵营，没有阵营区分则填-1

随机朝向：勾选后生成怪物朝向随机，若要精确控制怪物朝向可通过刷怪点上的蓝色箭头控制（如下图）
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KfMxOimage.png)
最小生成数量：该刷怪点同一波次最小刷新的怪物数量
最大生成数量：该刷怪点同一波次最大刷新的怪物数量

单次刷怪数量：单次刷怪只数

### 怪物刷新器

场景中只有一个刷怪管理器，用来管理每个刷新器。
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OQRfgimage.png)

在刷怪管理器中，可以配置当前关卡的怪物总波次，以及每一波具体刷新的敌人内容。每一波都会独立配置对应的怪物信息和刷新方式。

示例中共配置了20波怪物刷新，并且每一波的结束条件均为该波所有怪物被击败后，才会进入下一波。
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/tciFLimage.png)

刷怪管理器启动方式：包括事件触发、关卡加载和手动调用三种方式
事件名：当刷怪管理器启动方式选择事件触发时监听的事件名
刷怪波次：将关卡内刷怪点分组管理，以不同刷怪波次按数组顺序依次触发各组刷怪点
刷怪点：在刷怪点蓝图中勾选了是否需要刷怪管理器属性的刷怪点，就会出现在这里的下拉列表中，可将该刷怪点配置在刷怪管理器的单个或多个波次中，对应的即是场景中对应刷新器的位置
覆盖怪物刷怪类型配置：勾选后将会以下方的怪物类型配置覆盖刷怪点自带的怪物类型配置，通常勾选。每个配置都会重载刷怪点上的原本内容。
波次开始条件：当前波次的开始条件，包括All Mob Die和Last Wave End两个选项，分别表示上一波怪物全部死亡和上一波怪物刷新完毕。
延迟开始时间：当满足当前波次开始条件时开始计时，在延迟时间后开始刷新这波次的怪物。

### 属性成长

在刷怪管理器的Lua代码中，会对每波次出生的怪物进行属性设置
```lua
function BP_SpawnerManager:ApplyMonsterAttributes(Pawn)
    local MonsterConfig = TopDownSurvivor:GetMonsterConfig(Pawn.CharacterName)

    if MonsterConfig == nil then
        return
    end

    for AttributeType, Value in pairs(MonsterConfig.Attributes) do
        local GrowthType = GetGrowthTypeFromAttributeType(AttributeType)
        local GrowthValue = 1

        local CustomGrowth = MonsterConfig.CustomWaveGrowth[self.CurrentWaveIndex]
        if CustomGrowth and CustomGrowth[GrowthType] then
            GrowthValue = CustomGrowth[GrowthType]
        else
            local GrowthPerWave = MonsterConfig.GrowthPerWave[GrowthType] or 0
            GrowthValue = self.CurrentWaveIndex * GrowthPerWave
        end

        -- GameLog.Log("BP_SpawnerManager:ApplyMonsterAttributes", string.format("AttributeType: %s, GrowthValue: %f", AttributeType, GrowthValue))
        UGCAttributeSystem.SetGameAttributeValue(Pawn, AttributeType, Value + GrowthValue)
    end
end
```


## 怪物掉落

### 怪物掉落配置表说明

当前怪物使用的是 UGC 内置掉落机制，只需配置 UGCDrop 表即可完成掉落控制。

UGCDropGroup 主要用于更复杂的掉落需求，例如在一次掉落中同时产出多组道具，或对大量掉落内容进行分组管理与复用。

在当前样例中，设计上限定每个怪物最多只会掉落一个道具，因此未使用掉落组表，而是直接通过 UGCDrop 进行配置。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/OtnFximage.png)

掉落ID：自定义，和怪物掉落组件对应即可
描述：物品描述，会在物品UI等位置显示
掉落类型：有概率和权重两种。主要区别是，概率掉落则可能无任何掉落，权重掉落则必然掉落。
随机次数：掉落时进行几次掉落随机计算
物品信息：该ID对应掉落物的配置
权重/概率：根据掉落类型填即可，概率的数值为【设定概率×10000】，即设定概率为10%，则在此处填1000
物品ID：对应物品编辑器中，需要掉落的物品的ID
物品数量Min/Max：若掉落该物品，则最小/最大掉落多少。若只掉落一个该物品，则任意一个配置填1即可

### 配置表生效

默认状态下，所有的怪物在掉落组件中都有下图样式的配置，表示死亡时会掉落这三种道具，掉落的具体规则由上述表格决定。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/e3Ko4image.png)

其中掉落组方案若无，则填-1。
掉落物类型有三种，在该玩法中，掉落物为玩家捡起自动使用的物品，所以选择Wrapper。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/0ZaZQimage.png)

Wrapper：一个个独立的地面可拾取物品

死亡盒子：生成一个死亡盒子，物品都进入死亡盒子

## BUFF

在当前玩法模板中，玩家可通过商店购买道具获得BUFF，BUFF可影响属性或赋予特殊能力；即时使用的掉落物同样通过BUFF实现生效；BUFF由技能编辑器配置，并通过物品编辑器与道具配置表完成与道具的绑定后生效。

### BUFF配置

详细buff配置方案，详见[Buff编辑器](https://developer.gp.qq.com/wikieditor/#/catalog/20087) 。

以空模板为模板创建一个BUFF。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/0S1C2image.png)
![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/rKODVimage.png)

UI信息：包含Buff名字、描述、图标，以及是否显示UI
阻碍Tag：当Buff挂载到角色身上时，若角色身上已有阻碍Tag中配置的Tag，则挂载失败
拥有Tag：当Buff成功挂载到角色身上时，角色将拥有这些Tag
打断Tag：当Buff成功挂载到角色身上时，角色身上的若拥有这些打断Tag，那么这些Tag及其关联的技能或Buff等将被移除
禁用Tag：当Buff成功挂载到角色身上时，角色身上若若拥有这些打断Tag，那么这些Tag及其关联的技能或Buff等将被移除，且后续拥有这些Tag的技能或Buff将不能挂载到角色身上.
合并条件：两个Buff能否合并成同一个Buff的条件
合并行为：两个Buff合并后的行为
最大堆叠次数：仅在合并行为为堆叠时显示，限制该buff堆叠的次数
堆叠类型持续时长计算方式：仅在合并行为为堆叠时显示，配置堆叠类型的持续时长
触发效果：效果包括修改属性和造成伤害，修改后的属性大于0才会修改成功
生效时长：单层Buff的默认持续时间，设置<0的值代表永久生效，Buff实际时长受合并效果影响
类型Tags：通过 GameplayTag 标记的Buff类型，适用于筛选和分类场景，遵循 GameplayTag匹配规则

### 样例BUFF说明

#### 道具BUFF

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/qMFrximage.png)

### 被动技能

#### 自动瞄准

这个被动技能会每隔0.1秒触发一次事件，该事件会依次执行以下两个动作：

1.选择目标
使用球形选取器检测周围目标，选取距离自身最近的10个对象，并按照距离由近到远进行排序。

2.持续调用函数
在技能 Lua 逻辑中，通过 SetControllerFaceTo 函数，使角色持续朝向已选目标中的第一个对象（即最近目标）。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/XXl5Qimage.png)

```lua
-- 将玩家角色始终朝向选择的目标
function Passive_AutoAim:SetControllerFaceTo()
    local Targets = self:GetSelectTargetActor(EPESkillSelectTarget.E_PESKILL_PickerType_AllTarget)
    -- GameLog.Log("AutoAim", string.format("Targets num: %d", #Targets))

    local Owner = self:GetOwnerActor()
    if Owner == nil or UGCPlayerPawnSystem.HasPawnState(Owner, EPawnState.Dead) then
        return
    end

    local PlayerController = UGCGameSystem.GetPlayerControllerByPlayerPawn(Owner)
    if PlayerController == nil then
        return
    end

    if Targets == nil or #Targets == 0 then
        return
    end

    for _, Target in ipairs(Targets) do
        local bTargetDead = UGCPersistEffectSystem.HasDynamicState(Target, "PawnState.Dead")

        if not bTargetDead then
            -- GameLog.Log("AutoAim", string.format("Face to target: %s", UGCObjectUtility.GetObjectName(Target)))

            local TargetLocation = Target:K2_GetActorLocation() 
            local OwnerLocation = Owner:K2_GetActorLocation()
            local Direction = UGCMathUtility.SubtractVector(TargetLocation, OwnerLocation)
            Direction.Z = 0
            Direction = UGCMathUtility.Normal(Direction)
            local Rotation = UGCMathUtility.Conv_VectorToRotator(Direction)
            UGCPlayerControllerSystem.SetControlRotation(PlayerController, Rotation)
            return
        end
    end
end
```

若有类似的需求，可以参考此代码实现

### 近战武器技能

为了可以实现武器攻击时候的范围可以和角色自身的攻击范围对应上这个机制，需要在这个技能的代码中动态读取角色当前对应的属性，然后设置给技能实例中指定的变量

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/kRQfMimage.png)

```lua
function Melee:OnEnableSkill_BP()
    Melee.SuperClass.OnEnableSkill_BP(self)

    local OwnerActor = self:GetOwnerActor()

    self.AttackRange = UGCAttributeSystem.GetGameAttributeValue(OwnerActor, E_CharacterAttributeType.AttackRange)
    
    self.AttackRangeChangeDelegate = UGCAttributeSystem.AddGameAttributeChangedDelegate(OwnerActor, E_CharacterAttributeType.AttackRange,
        function (Owner, AttrName, NewValue)
            self.AttackRange = UGCAttributeSystem.GetGameAttributeValue(OwnerActor, E_CharacterAttributeType.AttackRange)
        end
    )
end

function Melee:OnDisableSkill_BP()
    Melee.SuperClass.OnDisableSkill_BP(self)

    local OwnerActor = self:GetOwnerActor()

    UGCAttributeSystem.RemoveGameAttributeChangedDelegate(OwnerActor, E_CharacterAttributeType.AttackRange, self.AttackRangeChangeDelegate)
end
```

然后在范围选取器中，将选取器半径配置为“属性获取”，这样在每次技能触发时，选取器都会实时读取玩家当前的攻击范围属性，使技能的实际攻击半径始终与玩家当前攻击范围保持一致。

枪手使用的枪械打出的抛体有两个关键属性

 - 穿透

 - 弹射

这两个属性会影响抛体的运动轨迹，假如抛体发射的时候穿透次数为3，弹射次数为2，则抛体在命中对象时候会：

 - 监测当前是否仍有穿透次数剩余，有则穿过敌人
 - 若当前没有穿透次数剩余，则将当前的飞行轨迹替换为“目标弹射轨迹”，弹射次数初始化就是2
 - 监测当前是够仍有弹射次数剩余，有则执行弹射相关的逻辑
 - 若当前没有剩余的弹射次数，则执行后续的弹射完成事件

使用这个抛体修改器就可以让一个抛体既有穿透属性又有弹射属性。

## 怪物属性

怪物表用来配置该玩法中的每个怪物属性。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/DPs27image.png)

DropExp：该怪物被击杀时，玩家可获取的经验值（存在多名玩家时，每个人获得相同的该数额的经验值）
DropCoin：该怪物被击杀时，掉落的金币数量
Attributes：属性
BaseHealth ：怪物基础生命值
MoveSpeedScale：怪物移动系数
Attack：基础攻击力
CastSkillSpeed：施法速度（实际在该模板中，会在技能编辑器中配置怪物的技能及释放频率）
AttackFrequency：攻击频率
Defence：怪物防御
HitBackResistence：击退抗性（百分比）
GrowthPerWave：每波次成长，即每一波怪物的属性成长值，均为加值
HealthGrowth：生命值成长
MoveSpeedGrowth：移动速度成长
AttackGrowth：攻击力成长
DefenceGrowth：防御成长
DropExpGrowth：掉落经验值成长

CustomWaveGrowth：特定波次成长，即在某一特定波次的属性成长，可按需配置

属性管理器：用于控制其他额外属性

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/wuiFrimage.png)

对于```AttackFrequency```，攻击频率的实际生效公式为：当前武器攻击频率 = 武器默认攻击速率 ÷ 攻击频率。

其中，武器默认攻击速率在两个职业对应使用的武器（物品编辑器 / 技能编辑器）中进行配置；攻击频率为玩家属性配置表中的数值；当前武器攻击频率则是最终实际生效的结果。

若希望玩家的某一属性不会低于指定值，则可以在属性管理器中配置该属性的最小值。

# 商店

在本样例工程中，玩家每次升级时都会触发商店界面。在商店内，玩家可以使用击杀敌人获得的金币购买道具，这些道具会为玩家提供各种增益效果。

## 商店内容配置表

玩法工程中的商店刷新，本质上使用的是通用的怪物掉落表```UGCDrop```。每次商店出现后，会根据当前波次选取对应的掉落组进行随机，并通过权重来决定本次商店刷新出的具体物品。如下图所示，通过在表中配置多种道具并设置不同权重，即可在特定波次中实现商店刷新结果具有一定随机性的效果。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/RlyX5image.png)

另外，商店中的道具需要额外配置一个表格，用于记录其价格以及获取后对角色施加的 Buff、技能等属性，此处使用的是```ItemConfig```表。本玩法工程中所有道具的基础属性均通过该表进行统一控制。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/1Dw33image.png)

ItemID：对应物品编辑器中的对应物品ID
ItemName：通过UI展示出的道具名称
Price：若该物品会在商店中出现，则需要配置购买所需的价格
Buffs/Skills：该道具对应的效果/技能

技能相关道具有额外配置项，用于配置多次获取该技能道具后，该技能可获得的升级内容：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/39UQ1image.png)

CDReductionPerStack：每层CD缩减（秒）
BounceCountPerStack：每层弹射次数增加
