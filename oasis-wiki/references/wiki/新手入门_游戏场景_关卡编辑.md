#  超大地图关卡加载方式：World Composition（基于大地图模板建立的地图）
**World Composition：** 和平大地图模板地图加载方式（海岛、沙漠、雨林、雪地、山谷）。

<br>

## 超大地图加载（World Composition）

### 1.  什么是超大地图加载：

- World Composition。适用于超大地图多关卡管理，这种模式下的关卡加载方式可以大大节省内存的占用；
- 激活World Compositiom加载后，该关卡就变成了Persistent,并且, 会遍历该关卡所在目录及其子目录的所有关卡文件, 将他们作为子关卡自动添加到该关卡下。

### 2.  激活方式：
- 可通过菜单栏中“设置”→“世界设置”窗口进行管理，勾选Enable World Composition<br>
![企业微信截图_16866492522079.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866492522079.png)

### 3.  子关卡的增减

- A.常规编辑：创建 编辑 保存都通用基本操作<br>
- B.模板原有子关卡：<span style="color: #ff0000">**和平大地图模板中原有的子关卡，都不可单独编辑修改**，</span>但是可以选择不加载某一关卡。不想加载的子关卡需要填入项目设置的黑名单中<br>
![企业微信截图_16915659466030.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16915659466030.png)

### 4.  子关卡的加载方式

World Compositiom的模式下，子关卡有两种加载方式：按距离加载&蓝图加载
- 设置子关卡加载方式：关卡编辑窗口选中要编辑的关卡，选择“分配给层”，会出现关卡加载方式选项<br>
![企业微信截图_16866492696915.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866492696915.png)

- 按距离加载：可设置不同的加载距离，当玩家和该关距离满足条件时，关卡就会被加载
  自定义关卡加载距离：在关卡面板中点击“世界构成”，然后可以按着自己需求新建不同距离的加载层级，
  Streaming distance:100=1M<br>
![企业微信截图_16866492826732.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866492826732.png)

- 蓝图加载：把关卡加载方式设置为LoadByBP,然后自己写加载逻辑
  范例：场景中摆放一个名叫“LoadLevel” 的触发器actor，当有玩家进入该触发器范围的时候，就加载名为“NewMap01”的关卡，并且显示“NewMap01”关卡下的所有内容。玩家退出触发器范围后，卸载掉“NewMap01”关卡，示例如下：
	
```lua
local LoadLevel = {}

function LoadLevel:ReceiveBeginPlay()
    ugcprint("LoadLevel:ReceiveBeginPlay");
    if self:HasAuthority() == true then
        self.Box.OnComponentBeginOverlap:Add(self.OnBeginOverlap, self);
        self.Box.OnComponentEndOverlap:Add(self.OnEndOverlap, self);
    end
end


function LoadLevel:OnBeginOverlap()
    GameplayStatics.LoadStreamLevel(self, "NewMap01", true, false)
end

function LoadLevel:OnEndOverlap()
    GameplayStatics.UnloadStreamLevel(self, "NewMap01")
end

-- end

-- end
-- function LoadLevel:ReceiveEndPlay()
 
-- end
return LoadLevel;
```

#  一般地图子关卡管理（基于空模板建立的地图）

**非World Composition：** 以空模板为基础建立的工程，一般用于非超大地图的加载。

<br>

## 一般地图子关卡加载（未激活World Composition）

### 1.  使用范围：

- 一般地图，没有太大的占地面积或者太多的物件层级，就可用此常规方式管理地图；

### 2.  设置方式：

- 可通过菜单栏中“设置”→“世界设置”窗口进行管理，默认未勾选Enable World Composition<br>
![企业微信截图_16866493081305.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866493081305.png)

### 3.  子关卡的增减

- 常规编辑：创建 编辑 保存都通用基本操作

### 4.  子关卡的加载方式

该模式下有两种子关卡加载方式：永远加载&蓝图加载

- 设置子关卡加载方式：关卡编辑窗口选中要编辑的关卡，选择“改变动态加载方式”，会出现两个的关卡加载选项<br>
![企业微信截图_16932927761103.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/K1k3B%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16932927761103.png)
- 总是加载：就是会一直跟随主关卡一起加载
- 蓝图加载：把关卡加载方式设置为LoadByBP,然后自己写加载逻辑<br>
  范例：参见 [超大地图关卡加载方式](https://developer.gp.qq.com/wikieditor/#/catalog/279)

#  关卡编辑基础

## 关卡简介

- **关卡：** 绿洲启元编辑器中，所有玩家看到的对象和交互的对象，我们都可以保存在一个“世界“”里，这个“世界”我们称之为关卡，关卡中可以保存编辑器中大部分的资源类型。
- **永久性关卡** ：工程的主关卡，每个项目始终都会有一个永久性关卡。
- **子关卡** ：也是独立的关卡文件，可通过不同方式与主关卡建立连接管辖。

<br>

## 关卡初识

管理多个关卡：可通过【关卡】窗口进行管理，在菜单栏中可找到。

### 1. 创建子关卡

- 新建：单击 【关卡】下拉菜单，然后选择 【新建关卡】，可新建一个子关卡。
- 添加：单击 【关卡】 下拉菜单，然后选择 【添加现有关卡】，可添加一个现有关卡为子关卡。
- 拆分：先在场景大纲中选中要拆分的物件，然后单击 【关卡】 下拉菜单，然后选择 【新建带有选中物体的关卡】，可为选中的物体们创建一个新的子关卡。

![企业微信截图_16866484318950.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866484318950.png)

### 2. 编辑保存子关卡

双击可选中要编辑的子关卡，才可对其进行编辑，蓝色为当前选中关卡。下图蓝色为选中态，有小铅笔表示内容有修改，需要保存。

### 3. 关卡细节查看

关卡信息查看面板：可以设置子关卡的旋转 位置 和显示颜色等

![企业微信截图_16866484601146.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_16866484601146.png)

### 4. LevelBounds

 LevelBounds代表关卡的边界。会根据当前关卡中所有物体的的位置，自动生成该关卡的范围边界，用来计算该关卡大小，不可人为修改LevelBounds大小，但是改变关卡中物体的位置。
 
![企业微信截图_168664847056.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E6%88%AA%E5%9B%BE_168664847056.png)

### 5. 子关卡编辑器内的显示和隐藏

先选中关卡，选中的关卡为蓝色。双击选中要编辑的子关卡，所有的新增编辑都只会对当前选中的子关卡生效（所有新加入场景的资源，会被保存到当前选中的子关卡中）。被load出来的关卡为白色，在编辑器内可见，未被load出来的关卡为灰色，在编辑器内不可见。
