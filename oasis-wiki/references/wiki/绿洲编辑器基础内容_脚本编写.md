#  游戏结束时的逻辑处理

游戏的战斗阶段结束后，需要将玩家的结算数据上报，而后关闭服务器并断开客户端连接，这是游戏结束时必须要添加的逻辑；如果没有正确处理这部分逻辑，DS服务器超过有效时长将被异常关闭，导致玩家数据未被正确保存而丢失，以及客户端异常等表现。

<br>

## DS关闭时机

每个DS进程有固定的有效存活时间，超过时长将被强制杀进程以关闭DS，因此每个玩法需要确保在有效时间内被正确关闭。

> DS默认有效时长为30分钟，如有需要可联系绿洲官方运营人员提出调整申请

**DS剩余时长** 

开发者可以调用 [``GetDSRemainingTime``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=GetDSRemainingTime) 查询DS关闭前的剩余时长，以及时处理结束前的收尾逻辑。

```lua
local RemainTime = UGCGameSystem.GetDSRemainingTime()
print(string.format("当前DS剩余关闭时间：%s 秒", RemainTime))
```

**DS关闭通知**

[``SetDSCloseNotify``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=SetDSCloseNotify) 提供了监听DS关闭前通知的机制，通过此函数可以注册希望通知的时间点，系统将在指定的时间点通过 ``UGCGenericMessageSystem.UserDefinedMessages.UGC.UGCDSShutDownManager.DSCloseNotify`` 事件进行广播通知，在监听事件的回调中处理游戏阶段的逻辑。

```lua
-- 注册通知点，离关闭前60秒将触发事件
UGCGameSystem.SetDSCloseNotify({60})

-- 监听关闭通知事件，执行数据存档
UGCGenericMessageSystem.ListenGlobalMessage(
	UGCGameSystem.GameState,
	UGCGenericMessageSystem.UserDefinedMessages.UGC.UGCDSShutDownManager.DSCloseNotify,
	self,
	function()
		UGCPlayerStateSystem.SavePlayerArchiveData(UID, self.PlayerData)
  end
)
```

## 结束处理步骤

### 1. 显示结算界面

游戏对局结束后，通常需要弹出独立的结算界面，让玩家可以清楚了解到获得的分数与奖励，同时界面中需要保留返回大厅的功能，可以让玩家顺利返回和平大厅。

返回大厅功能形式，可以参考以下两种：

- 返回大厅按钮：在结算界面，设置返回大厅按钮，玩家点击按钮后，即可顺利返回大厅。
- 倒计时返回大厅(可选)：进入结算界面后，启动倒计时程序，在界面中显示倒计时时间，时间结束后，玩家自动返回大厅。

![4264ecfc1d3906ab4ea51cf95be653f.jpg](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/Y3r444264ecfc1d3906ab4ea51cf95be653f.jpg)

---

### 2. 触发结束游戏事件

当游戏对局结束后，需要处理发送结算以及关闭DS服务器等逻辑，为此需要创建一个用于监听游戏结束的触发器。

首先创建自定义事件，点击模式编辑器，点击编辑事件，新建一个事件，取名为GameEnd。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/pMlxHimage.png)

而后，创建一个新的流程触发器，取名为结束游戏，事件类型(Event Type)选择刚刚创建好的自定义事件【GameEnd】。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KkteIimage.png)

当游戏对局结束后，使用编辑器的事件接口来触发【GameEnd】事件
> 接口：LuaQuickFireEvent([事件名称], [发出事件的对象], [事件参数]);
	
在确定对局结束的代码逻辑中，调用此接口，以下是示例：

``` lua
if UGCGameSystem.GameState.CurrentRound >= self.MaxRound then --当前记录的对局数已经达到最大对局数，游戏结束
	LuaQuickFireEvent(“GameEnd”, self); -- 触发【GameEnd】事件
end
```

在触发【GameEnd】事件后，将自动执行绑定在事件之下的动作（Action）。

---

### 3. 结算与退出DS

关闭DS前需要将每个玩家的数据发送给后台进行结算上报，该数据会影响玩家完成绿洲启元相关的任务和数据统计，模式编辑器将发送结算与关闭DS的逻辑封装成一个动作脚本 ``UGC请求退出DS``，开发者仅需在触发【GameEnd】事件之后，指定执行该动作即可。

> 无需再单独调用 [``SendPlayerSettlement``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=SendPlayerSettlement) 做结算上报

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/vS6h1image.png)

---

### 4. 断开客户端链接

DS关闭后，需要同步关闭客户端对服务器的长链接检测，否则玩家客户端会弹出无法连接到服务器的报错信息。可以在客户端监听OnGameEnd事件并调用 [``DisconnectClient``](https://developer.gp.qq.com/api/#/searchContent/UGCGameSystem?classDetailShow=true&path=class%2Fdetail%2F%E5%92%8C%E5%B9%B3%E5%85%A8%E5%B1%80%E6%8E%A5%E5%8F%A3%2F%E5%9F%BA%E7%A1%80%E5%8A%9F%E8%83%BD%2FUGCGameSystem.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UGCGameSystem&autoJump=DisconnectClient)。

代码示例：

``` lua
-- 当监听到游戏结束事件时，触发断开客户端链接
function MainUI:OnGameEnd(GameResultData)
    ugcprint("MainUI:OnGameEnd GameResultData: ", tostring(GameResultData))
		
		-- 断开客户端链接
		UGCGameSystem.DisconnectClient()

    -- TODO 其他逻辑
end
```

## 特殊情况：无存活玩家

在DS服务器中，如果所有玩家全部处于死亡状态（无存活Pawn），服务器会开启超时检测：若在一定时间内无有效存活玩家，将会强制关闭。

如果玩法设计中存在所有玩家全部死亡，且游戏流程仍需继续的场景，则需要在UGCGameMode中修改检测时间，以避免DS服务器被强制关闭而导致游戏被中断，如图所示：

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/eZIlximage.png)

``Check Period`` 默认设定的时间为120s，开发者可以根据需要修改设定时间

**注意**
> 虽然可以使用这个方法来避免DS服务器被强制关闭，但我们还是建议，在游戏玩法设计时，尽可能避免出现所有玩家死亡或其他极端的情况，原则上尽可能不要修改检测超时的参数。

# 绿洲启元Lua脚本开发指南

绿洲启元编辑器使用Lua作为脚本语言，可以使用大部分Lua库和自带接口。但Lua脚本和引擎参数和接口交互，有一套自己的规则，下面将逐步介绍这些规则。

<br>

## 基础操作

### 1. 基础语法说明

- 对象成员属性访问使用 **点(.)** 访问；
- 对象成员属性修改使用 **点(.)** 访问后在 **等号(=)** 赋值即可；
- 对象成员方法使用 **冒号(:)** 调用；
- 类静态方法使用 **点(.)** 调用；

<details open>
<summary> <font face="楷体">示例：基础操作</font> </summary>

```lua
-- 类静态方法调用,大部分UE4全局类，如UGameplayStatics都可以直接把前缀删除后直接使用。
local PlayerPawn = GameplayStatics.GetPlayerCharacter(self,0) 
-- 对象属性访问，可以访问该对象C++/蓝图/Lua属性。
local PlayerName = PlayerPawn.PlayerName	
-- 对象属性修改，可以修改该对象C++/蓝图/Lua属性。
PlayerPawn.PlayerName = "Test"
-- 对象方法调用：使用默认值参数。
PlayerPawn:K2_SetActorLocation(Vector(0,0,0)) 	
--PS：由于C++的默认参数是编译期生效，Lua脚本是运行时生效，所以无法直接使用C++接口默认值，这里的默认值使用0x0，number型为0，bool类型为fasle，指针为nil，如果是结构体那么内部参数按照以上默认值规则。
```

</details>

### 2. 可访问数据说明

- 绿洲启元编辑器中Lua导出/反射都是通过UE4反射系统实现，因此只能访问 UProperty 数据和 UFunction 方法。
- BP中属性说明:
  
  - BP中变量/方法/代理列表中的数据都可在lua中访问;
  - BPActor中Component默认都是变量，即都可以在lua中访问;
  - BPUMG中WidgetTree中的子控件: 勾选`IsVariable`则会显示在变量列表中，可在lua中方位；未勾选，则变量列表中没有，也不可在lua中访问；

### 3. 可重写函数说明

- C++中 `UFUNCTION `宏中有`BlueprintNativeEvent`和`BlueprintPure`标记为可重写函数。
- 蓝图中定义的函数均为可重写函数。

<details open>
<summary> <font face="楷体">示例：重写函数示例</font> </summary>

```lua
local UGCGameMode = {}; 
function UGCGameMode:ReceiveBeginPlay()
    self.SuperClass.ReceiveBeginPlay(self)	--作用等同于C++的Super::ReceiveBeginPlay()
end
```

</details>

### 1. 数据类型说明

lua访问UE数据时：基础类型数据直接值传递；复合数据类型做了自定义封装，达到成员变量的引用放访问效果，并且能继续调用数据类型的成员方法.

### 2. 使用枚举

使用方式：枚举名 点 枚举值：`Enum.EnumItem`，该使用方式仅限于C++声明的枚举。

<details open>
<summary> <font face="楷体">示例：枚举</font> </summary>


</details>

### 3. 使用结构体

- lua中访问到的结构体为 `userdata`(带元表)，引用传递类型数据;
- 访问结构体属性使用基础语法访问即可;
- lua中给结构体赋值兼容 `userdata`类型(推荐)和 `table`类型(兼容支持) 赋值;
  - table 类型赋值时：Key为字段名, Value为字段值; 可部分赋值， 未赋值字段使用内存`0x0`填充;

<details open>
<summary> <font face="楷体">示例：数据类型-结构体</font> </summary>

```lua
local object = CreateUIWidget(bp_lua, "/Game/Demo/DemoUI.DemoUI_C");
local t1 = object.ColorAndOpacity;
local t2 = object.ColorAndOpacity.R;
local t3 = t1.R;

print(type(t1));        -- userdata

object.ColorAndOpacity = t1;                    -- 使用 userdata 类型赋值;
object.ColorAndOpacity = {R=1, G=2, B=3, A=1};  -- 使用table类型赋值; = FLinearColor(1,2,3,1);
object.ColorAndOpacity = {R=1, A=1};            -- 使用table类型赋值; = FLinearColor(1,0,0,1);
```

</details>

### 5. 使用代理

- 代理分为单播代理和多播代理;
- 单播代理：只能绑定一个回调函数; 多次绑定会覆盖，C++声明前缀为`DECLARE_DELEGATE`，如Button.OnMouseButtonDownEvent
  - 添加绑定：`Bind(CallBackFunc [, param])`;
  - 移除绑定：`UnBind()`;
  - 触发代理：`Execute(...)`;
- 多播代理：可绑定多个回调函数; 相同的绑定会自动去重，C++声明前缀为`DECLARE_DYNAMIC_MULTICAST_DELEGATE`，如Button.OnClicked
  - 添加绑定：`Add(CallBackFunc [, param])`:
    - `CallBackFunc` 作为Key: 同Func不同param会覆盖去重;
  - 添加绑定：`AddInstance(CallBackFunc[, param])`:
    - `CallBackFunc && param` 共同作为Key:同Func不同Param不会去重; 只有同Func同Param才会去重；
  - 触发代理：`Broadcast(...)`; (此处参数为代理的声明参数)
    - 移除绑定： `Remove(CallBackFunc [, param])`  or  `RemoveAll()` ;
    - 触发代理： `Broadcast(...)` ; (此处参数为代理的声明参数)
- 创建代理：创建代理这里主要用于SetTimer，代理的生命周期跟随Object，如果频繁的创建需要注意手动销毁；
  - 创建绑定：`ObjectExtend.CreateDelegate(Object, CallBackFunc)`
    - 删除代理：`ObjectExtend.DestroyDelegate(Delegate)`

<details open>
<summary> <font face="楷体">示例：数据类型-代理</font> </summary>

```lua
---@class MainUI_C:UserWidget   
---@field TestButton UButton            --该UMG有一个TestButton的按钮控件
local MainUI = {}; 
function MainUI:Construct()
    self.TestButton.OnMouseButtonDownEvent:Bind(self.TestButton_OnMouseButtonDown, self)   --绑定TestButton_OnMouseButtonDown函数
    self.TestButton.OnMouseButtonDownEvent:Bind(                                           --绑定匿名函数，单播委托此处绑定会直接覆盖上次绑定，这里仅做示范
        function(MyGeometry,MouseEvent) 
            print("MainUI:TestButton_OnMouseButtonDown")
        end, self)  

    self.TestButton.OnMouseButtonDownEvent:UnBind()                     -- 移除绑定
    self.TestButton.OnMouseButtonDownEvent:Execute(nil, nil)            -- 触发代理（控件事件不需要主动触发,此仅做用法示例）
```

## 方法调用

方法基础操作参考上文中【基础语法说明】板块， 注意成员方法与静态方法调用方式即可，普通的参数正常使用即可，无需特别注意。此部分主要针对特殊类型和标记的参数(引用/const等)。

- 方法返回值指：1.C++方法的return; 2.蓝图方法的Outputs;
- 返回值都是值拷贝方式返回， 即便声明为引用返回也是拷贝方式;
  - 拷贝是指对数据进行内存复制，然后返回复制的内存数据；
  - UObject类型实际都是指针类型UObject*:，返回指针数据值，不会复制UObject整体;
- 非const引用参数会作为返回结果输出，依次位于return值后面;

<details open>
<summary> <font face="楷体">示例：方法返回值</font> </summary>

```c++
// C++
// 简要声明:忽略各种标签宏说明等说明;
class UDemoWidget : public UUserWidget
{
    int32 Param;
    int32 GetParam()      { return Param; }
    int32& GetParamRef()  { return Param; }
	bool GetDoubleParam(int32& OutParam1, int32& OutParam2)
	{ 
		OutParam1 = Param + OutParam1;
		OutParam2 = Param + OutParam2;
		return true;
	}
};
// /Game/Demo/DemoUI 继承于 UDemoWidget 创建;
```

```lua
-- lua
local DemoWidget = CreateUIWidget(bp_lua, "/Game/Demo/DemoWidget.DemoWidget_C");

local Param = DemoWidget:GetParam();
local ParamRef = DemoWidget:GetParamRef();      -- 等价与 GetParam 调用;
ParamRef = 123;   -- 声明引用也是拷贝形式返回：赋值只更新lua变量值, 不会更新DemoWidget中的Param;

local OutParam1,OutParam2;
local Result;
Result,OutParam1,OutParam2 = DemoWidget:GetDoubleParam(OutParam1,OutParam2)	 --非const引用参数会作为返回结果输出,依次位于return值后面
```

##  重点说明
	
1. （内存）lua 引用的 UObject 变量，请在销毁前手动 set nil，缓存到lua后被UE销毁的对象，在lua访问到的是invalid Object，而不是nil，判断的时候要用UE.IsValid。

```lua
local Actor = SpawnActor(...)
Actor:K2_DestroyActor()
--这里建议销毁后手动设置Actor = nil
if Actor == nil then end   --Actor不为nil
if UE.IsValid(Actor) == false then end  --UE.IsValid(Actor)为false
```
	
2. （功能）lua调用C++方法时默认参数请主动填入，不要省略，除非需要传入的值为`0x0`。

```c++
void CPPFunction(int Param = 1);
```

```lua
CPPFunction();--这里会使用0x0的值，也就是Param会是0
CPPFunction(1); --√
```

5. （功能）totable 仅用于调试及特殊需求，Shipping下仅支持数组容器操作，会导致调试功能和真机运行功能不符。
6. （性能）lua中频繁传递结构体参数到UE4，尽量使用结构体而非table，会有额外的转换开销。

```lua
void CPPFunction(Vector Param);
......
CPPFunction({X=1,Y=2,Z=3});		--×
CPPFunction(Vector.New(1,2,3));	--√
```

8. （性能）请缓存续频繁使用到的数据，每次访问C++/蓝图参数都有额外的查询开销。
9. （性能）lua容器缓存UE管理的c++指针数组时，会存在指针被UE释放掉的情况，从而引发DS崩溃，避免使用Copy的方式对指针数组进行缓存，可以使用ToTable的方式缓存，**<span style="color:red;">以下为错误案例：</span>**
```lua
function BP_Worship:RecordBeginAreaPlayer(CheckActorArray)
	self.TotalBeginAreaPlayer = CheckActorArray.OverlapActorList:Copy()
end
```

# Actor实例路径使用

Actor 实例路径功能用于在UGC编辑器中快速引用场景内的指定实例。开发者可以通过复制实例路径，并在运行时基于该路径查找对应的 Actor ，从而高效完成对象引用与操作。

在世界大纲或场景中可被选中的对象，均属于 Actor 实例。

<br>

## 功能概述

Actor 实例路径提供了一种统一、简洁的方式，用于在编辑器与运行时之间建立实例引用关系。
相比传统手动维护引用的方式，该功能具有以下优势：

- 支持Lua配置直接引用：在配置中直接填写实例路径，无需额外绑定
- 便于关卡脚本定位对象：可快速查找预设点位、触发器、机关等
- 降低维护成本：场景调整时无需同步修改代码引用逻辑，减少手工维护引用关系的成本，提高开发效率
- 路径统一规范：可读性强，便于管理与排查问题

## 功能使用

### 功能说明

在UGC开发中，常见的实例引用方式包括：

- 手动维护对象引用
- 编写复杂查找逻辑（如遍历、标签匹配等）
- 场景变更后需同步修改代码

Actor实例路径机制提供了更高效的替代方案：

- 编辑器中右键选中实例后一键复制路径
- 运行时通过路径直接定位 Actor
- 无需维护额外引用关系

### 路径格式

实例路径采用统一格式：
`<LevelName>.<ActorInstanceName>`

示例：
`"UGCmap.BP_SpawnerElite4"`

字段说明：
|组成部分|说明|示例|
|:-:|-|-|
|`LevelName`|关卡 / 世界名称|`UGCmap`|
|`ActorInstanceName`|实例在关卡中的对象名|`BP_SpawnerElite4`|

1.在编辑器场景视图中中选中目标 Actor
2.右键打开对象的上下文菜单
3.点击获取示例路径

### 运行时使用方式

在运行时通过`UGCActorComponentUtility.GetActorByActorInstancePath`来获取实例。
```lua
-- 通过实例路径获取 Actor
local Actor = UGCActorComponentUtility.GetActorByActorInstancePath("UGCmap.test_8")

-- 验证 Actor 有效性并使用
if Actor and UGCObjectUtility.IsObjectValid(Actor) then
    ugcprint_concat("Find Actor Success", Actor:GetName())
    -- 后续可以对该 Actor 进行操作
end
```

### 使用注意事项

- 有效性校验：使用`UGCObjectUtility.IsObjectValid()`
- 多子关卡支持：若地图中有多个子关卡，接口会遍历所有关卡
- 对象生命周期：确保 Actor 已完成创建与初始化

## 典型应用场景

场景一：Lua 配置引用固定实例
```lua
-- 在 Lua 配置中直接引用场景中的目标对象
local TargetActorPath = "UGCmap.Dynamic_ShootingRange_Target5"
local TargetActor = UGCActorComponentUtility.GetActorByActorInstancePath(TargetActorPath)

if TargetActor then
    -- 对目标对象进行配置或操作
    TargetActor:SetActorLocation(Vector.New(100, 100, 100))
end
```

场景二：查找关卡检查点
```lua
-- 关卡脚本通过路径查找预设的点位
local CheckpointPath = "UGCmap.CheckPoint_1"
local CheckpointActor = UGCActorComponentUtility.GetActorByActorInstancePath(CheckpointPath)

if CheckpointActor then
    -- 检查玩家是否到达检查点
    local Player = UGCPlayerController.GetPawn()
    if Player then
        local Distance = Player:GetDistanceTo(CheckpointActor)
        if Distance < 10.0 then
            -- 到达检查点，触发相关逻辑
        end
    end
end
```

场景三：触发器/机关控制
```lua
-- 通过路径查找场景中的触发器或机关
local TriggerPath = "UGCmap.Trigger_Gate1"
local TriggerActor = UGCActorComponentUtility.GetActorByActorInstancePath(TriggerPath)

if TriggerActor then
    -- 监听触发器事件或进行激活操作
    TriggerActor:Activate()
end
```

场景四：批量处理实例
```lua
-- 批量复制多个实例路径后，在脚本中统一处理
local TargetPaths = {
    "UGCmap.Target1",
    "UGCmap.Target2",
    "UGCmap.Target3"
}

for i, Path in ipairs(TargetPaths) do
    local TargetActor = UGCActorComponentUtility.GetActorByActorInstancePath(Path)
    if TargetActor then
        -- 对每个目标进行统一操作
        TargetActor:SetActorScale3D(Vector.New(2, 2, 2))
    end
end
```
